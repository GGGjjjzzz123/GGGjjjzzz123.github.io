算法设计方法分类

贪心

独木舟——容量约束的配对问题——排序+贪心+双指针

  • 贪心为什么能解决配对问题：

贪心：先把最重的排好

若能配对，用最重+最轻，不会比其他搭配更差（交换论证）；若不能配对，最重+最轻都超重，那和任何人都超重，只能单独乘坐。

  • 思路：把体重从小到大排序，每次先排最重的和最轻的进行组队，若超重就单独乘坐。
  • 时间复杂度：sort排序O(nlogn)，双指针线性扫一遍O(n)，总复杂度O(nlogn)。

排队接水——最小化等待时间问题——排序+贪心（最短作业优先）

  • 思路：按接水时间从小到大排队，可使“完成时间总和”最小。

0+1（自己的等待时间+任务时间）自己的等待时间=前面任务时间

1+2（自己的等待时间+任务时间）自己的等待时间=前面任务时间

1+2+3（自己的等待时间+任务时间）自己的等待时间=前面任务时间

  • 所有人的花费的时间相加：（（（0+1）+1+2）+3+3）=10（自己等待时间+任务时间+前面人等待的时间之和）
  • 时间复杂度：sort排序O(nlogn)，一次性扫描求前缀和O(n)，总计O(nlogn)。
  • 空间复杂度：存放b[n]数组O(n)，sort排序O(logn)

活动安排问题——区间重叠数量的最大值问题——贪心+时间顺序扫描

  • 思路：在任意时刻，有多少个活动同时进行，这个最大同时进行数，就是所需教室的最少数量。把时间点当作“事件流”，每个活动产生两个事件：开始（活动数＋1）、结束（活动数-1）。把所有事件放在一起，从小到大扫描时间轴：遇到一个“开始”，当前教室占用+1，遇到一个结束，教室数-1，记录下整个过程中的cur的最大值，就是最少教室数。
  • 算法实现：两个数组+排序+双指针

时间复杂度：O（nlogn）

空间复杂度：O（n）

任务顺序执行

  1. 题目：有 N 个任务要依次执行。
    执行第 i 个任务时：
  • 计算阶段需要占用 R[i] 空间（临时占用）
  • 任务结束后会释放一部分临时空间，但结果要保留，结果占用 O[i] 空间（而且 O[i] < R[i]）

你可以决定任务的执行顺序。
问：为了能把所有任务都执行完，最少需要准备多少总空间（也就是执行过程中的最大占用峰值最小）。

输入

  • 第 1 行：N（2 ≤ N ≤ 100000）
  • 第 2 行到第 N+1 行：每行两个整数 R[i] O[i]
    （1 ≤ O[i] < R[i] ≤ 10000）

输出

  • 一行：最少需要的空间（最小峰值）
  1. 算法思想：算法思路是 贪心排序。

执行任务时，内存占用可以理解成两部分：已经做完的任务留下的结果空间 S（会一直累加），以及当前要做的任务临时需要的空间 R[i]。当你准备执行任务 i 时，瞬时峰值就是 S + R[i]；执行结束后临时空间释放，只留下结果空间，所以 S 变成 S + O[i]。因此我们要做的，是通过调整顺序让整个过程中最大值 max(S + R[i]) 尽量小。

关键在于：一个任务做完后，临时占用从 R[i] 降到 O[i]，它“释放掉”的空间是 R[i] - O[i]。这个释放量越大，说明这个任务越“先占很多、最后留下很少”，越适合早点做；否则拖到后面时 S 已经很大，再叠上它的 R[i] 更容易把峰值抬高。用两两交换可以证明：若 R[A]-O[A] >= R[B]-O[B]，把 A 放在 B 前面不会更差，所以最优顺序就是按 R[i]-O[i] 从大到小排序执行。排序后顺序扫描：维护 S，每步更新 ans = max(ans, S + R[i])，再令 S += O[i]，最终 ans 就是最少需要的空间。

  1. 代码
  1. 复杂度

时间复杂度：O(NlogN)

时间复杂度：O(N)

不重叠的线段

  1. 题目： 给你 N 条线段，每条有起点 S 和终点 E。你要从中选出尽可能多条，使得任意两条 不重叠。
    注意：端点相同不算重叠，也就是 [1,3] 和 [3,6] 可以同时选。

输入

  • 第 1 行：整数 N（2 ≤ N ≤ 10000）
  • 第 2 行到第 N+1 行：每行两个整数 S, E（-1e9 ≤ S,E ≤ 1e9）

输出

  • 一个整数：最多能选的线段条数
  1. 算法思路：要选最多条不重叠线段，最经典的贪心策略是：优先选择“结束最早”的线段。直觉是：结束越早，给后面留下的空间越大，就越容易再塞进更多线段。实现时先把所有线段按终点 E 从小到大排序，然后从前往后扫：维护上一次选中线段的终点 lastEnd，遇到一条线段如果 S >= lastEnd（端点相接允许），就选它并更新 lastEnd = E，计数加一。这样最终得到的数量就是最大值。
  2. 代码
  1. 复杂度

时间复杂度：O(NlogN)

空间复杂度：O(N)

  1. 题目：某个幼儿园里，有 5位小朋友编号依次为 1,2,3,4,5 他们按照自己的编号顺序围坐在一张圆桌旁。他们身上有若干糖果，现在他们玩一个分糖果游戏。从 1 号小朋友开始，将自己的糖果均分成 3 份（如果有多余的糖果，就自己立即吃掉），自己留一份，其余两份分给和他相邻的两个小朋友。接着 2,3,4,5 号小朋友也这样做。问一轮结束后，每个小朋友手上分别有多少糖果。

输入：

输出：

  • 第 1 行：一轮结束后 1~5 号手里糖果数（空格分隔）

e. 背包问题

  1. 题目：在 N件物品取出若干件放在容量为 W 的背包里，每件物品的体积为 W1 ， W2 …… Wn （ Wi为整数），与之相对应的价值为 P1,P2, …… Pn （ Pi 为整数）。求背包能够容纳的最大价值。

其中 1≤N≤100 ， 1≤W≤10000 ，每个物品 1≤Wi,Pi≤10000 。

Input

第 1 行输入两个整数 N 和 W ；
第 2∼N+1 行，每行两个整数 Wi 和 Pi ，分别表示每个物品的体积和价值。

Output

输出可以容纳的最大价值。

  1. 算法思想：这题是典型的 0/1 背包动态规划。它的核心思想不是去枚举所有“选/不选”的组合（那是 (2^N) 级别会爆炸），而是把“组合选择”压缩成“容量状态”。我们用 dp[j] 表示：在当前已经考虑过的一部分物品中，背包容量上限为 j 时能取得的最大总价值。这样一来，不管有多少种选法，都只需要维护 (W+1) 个状态。

当处理第 (i) 件物品（体积 (w)，价值 (p)）时，对于每个容量 j，只有两种决策：不选它，则最优值还是原来的 dp[j]；选它（前提 j >= w），则意味着需要从容量 j-w 的最优结果转移过来，得到 dp[j-w] + p。两种选择取最大，就得到状态转移：dp[j] = max(dp[j], dp[j-w] + p)。这里 max 的意义就是：在“选”和“不选”这两条路里，保留价值更大的那条结果。

之所以容量 j 要倒着枚举（从 W 到 w），是为了保证“每件物品最多使用一次”。因为倒序时，dp[j-w] 还保持着“上一轮（没用当前物品）”的旧值，所以把 p 加进去相当于“只用一次第 i 件”。如果正序枚举，dp[j-w] 可能已经在同一轮被更新过，等于当前物品已经被用过一次，你再加一次就会把同一件物品重复使用，算法就会错误地变成“完全背包”。处理完所有物品后，dp[W] 就是容量不超过 W 时能取得的最大价值。

  1. 算法思想： 这题的关键观察是：如果把所有边按权值从小到大排序，那么一棵生成树的“最小边权”和“最大边权”在排序后的边表里一定对应某个区间 [i,j][i, j][i,j]（第 iii 条边的权值做最小，第 jjj 条边的权值做最大），并且生成树的所有边都必须落在这个区间里。于是我们可以枚举区间的左端点 iii（也就是假设生成树最小边权为 w[i]w[i]w[i]），然后从 iii 往右扩展 jjj，用并查集不断把边加入集合，直到图在 [i,j][i, j][i,j] 这批边里已经能够生成一棵生成树（即并查集合并出了 n−1n-1n−1 条有效边，整图连通）。一旦首次连通，这个 jjj 就是对当前 iii 来说能达到连通的最小最大边权，因此候选答案就是 w[j]−w[i]w[j]-w[i]w[j]−w[i]，取所有 iii 的最小值即可。整个过程用到的核心工具仍然是 并查集判环/合并，本质是 Kruskal 的“只选不成环的边”，只不过我们不是为了最小总权值，而是为了最小区间跨度。

  1. 复杂度

时间复杂度：O(m2)

空间复杂度：O(m+n)

  1. 题目：如题，给出一个有向图，请输出从某一点出发到所有点的最短路径长度。

Input

第一行包含三个整数 n,m,s，分别表示点的个数、有向边的个数、出发点的编号。

接下来 m 行每行包含三个整数 u,v,w，表示一条 u→v的，长度为 w的边。

Output

  1. 算法思想： 本题用堆优化 Dijkstra。我们维护数组 dist[i] 表示目前找到的从起点 sss 到 iii 的最短距离估计，初始化时 dist[s]=0，其它为无穷大。然后用一个小根堆（优先队列）反复取出当前距离最小的点 uuu：因为所有边权非负，所以一旦 uuu 以最小距离出堆，这个距离就是最终最短路，不会再被更小的路径更新。接着用 uuu 去松弛它的所有出边 u→vu\to vu→v：如果 dist[u] + w < dist[v]，就更新 dist[v] 并把新距离入堆。为避免堆中旧状态干扰，每次出堆时判断 d != dist[u] 就跳过。最终得到所有最短距离；仍为无穷大的点说明不可达，按题目要求输出 231−12^{31}-1231−1。

a. 回家

  1. 题目：小 H 在一个划分成了 n×m个方格的长方形封锁线上。 每次他能向上下左右四个方向移动一格（当然小 H 不可以静止不动）， 但不能离开封锁线，否则就被打死了。 刚开始时他有满血 6 点，每移动一格他要消耗 1 点血量。一旦小 H 的血量降到 0， 他将死去。 他可以沿路通过拾取鼠标（什么鬼。。。）来补满血量。只要他走到有鼠标的格子，他不需要任何时间即可拾取。格子上的鼠标可以瞬间补满，所以每次经过这个格子都有鼠标。就算到了某个有鼠标的格子才死去， 他也不能通过拾取鼠标补满 HP。 即使在家门口死去， 他也不能算完成任务回到家中。

地图上有五种格子：

<font style="color:rgb(199, 37, 78);">0</font>：障碍物。

<font style="color:rgb(199, 37, 78);">1</font>：空地， 小 H 可以自由行走。

<font style="color:rgb(199, 37, 78);">2</font>：小 H 出发点， 也是一片空地。

<font style="color:rgb(199, 37, 78);">3</font>：小 H 的家。

<font style="color:rgb(199, 37, 78);">4</font>：有鼠标在上面的空地。

小 H 能否安全回家？如果能， 最短需要多长时间呢？

输入：

  • 第一行：<font style="color:rgb(0, 0, 0);">n m</font>
  • 接下来 n 行：每行 m 个数字（0~4）表示地图

输出：

  • 如果回不了家输出 <font style="color:rgb(0, 0, 0);">-1</font>
  • 否则输出最短时间（最少步数）
  1. 算法思想：把每一个“状态”定义成：(x, y, hp) —— 站在坐标 (x,y)，当前血量 hp(1~6)。
    从一个状态走到上下左右的相邻格，相当于走一条边（时间 +1），所以可以用 BFS 求最短步数。

状态转移（从 (x,y,hp) 走到 (nx,ny)）：

  1. 不能越界、不能走到 0。
  2. 新血量先变成 hp-1；如果变成 0，直接死亡，这条路无效。
  3. 如果新格子是 4（鼠标），并且你没死（新血量≥1），则瞬间补满：hp = 6。
  4. 如果新格子是 3（家），只要你没死（新血量≥1），就算到家。

因为 BFS 按步数一层层扩展，第一次到达家时的步数就是最短时间。

  1. 复杂度

时间复杂度：O(n⋅m⋅6⋅4)=O(nm)

b. 马走日

  1. 题目：马在中国象棋以日字形规则移动。请编写一段程序，给定 n×m 大小的棋盘，以及马的初始位置 (x,y) ，要求不能重复经过棋盘上的同一个点，计算马可以有多少途径遍历棋盘上的所有点。

Input

  • 第一行：整数 T (T < 10)，测试组数
  • 接下来每组一行：n m x y
    • 0 ≤ x ≤ n-1
    • 0 ≤ y ≤ m-1
    • n < 6, m < 6（所以最多 5×5=25 格）

Output

  • 每组输出一行：一个整数，表示从 (x,y) 出发遍历全棋盘的路径总数（可能为 0）
  1. 算法思想：把棋盘看成一个图：每个格子是一个点，马能从一个格子跳到另一个格子就连一条边。题目要我们从起点出发，走一条路径，把所有点都走一遍且不重复——这就是“全覆盖路径”。

做法就是 DFS 回溯：

- 用 `vis[i][j]` 标记格子是否访问过
- 从当前格子出发，尝试 8 个“日”字跳到下一个未访问格

1. **如果下一步没路走但还没走完，直接返回**
2. **孤立点剪枝（简单版）**：如果存在某个未访问格子，它已经没有任何“未访问邻居”，那它想被访问只能靠“下一步从当前点直接跳过去”。如果当前点跳不到它，就必死，直接剪掉。

b. 逆序数

  1. 题目：在一个排列中，如果一对数的前后位置与大小顺序相反，即前面的数大于后面的数，那么它们就称为一个逆序。一个排列中逆序的总数就称为这个排列的逆序数。
    如 2 4 3 1 中， 2 1 ， 4 3 ， 4 1 ， 3 1 是逆序，逆序数是 4 。给出一个整数序列，求该序列的逆序数。

输入：

  • 第 1 行：整数 NNN（N≤50000N \le 50000N≤50000）
  • 第 2~N+1N+1N+1 行：序列元素 A[i]A[i]A[i]（0≤A[i]≤1090 \le A[i] \le 10^90≤A[i]≤109）

输出：

  • 1 行：逆序数（一个整数）
  1. 算法思想： 暴力做法是两层循环枚举所有 (i,j)，时间是 O(N2)，但 N=50000 会超时。归并排序的核心是：把数组分成左右两半分别排好序，然后再把两个有序段“合并成一个有序段”。关键点在合并时：如果当前左段指针指向的数是 L[i]，右段指向的数是 R[j]，当出现 L[i] > R[j] 时，因为左段本身已经有序，所以 **从 **L[i]** 到左段末尾的所有数都会 > ****R[j]**，它们每一个都和 R[j] 构成逆序对；因此这一瞬间可以一次性加上“左段剩余元素个数”。这样每一层合并是线性的，总体就是 O(Nlog⁡N)。

  1. 复杂度
