SQL SERVER的系统数据库和用户数据库

系统数据库

用户成功安装 SQL Server数据库（软件）后，打开该数据库时会发现系统会自动建立 master 、 msdb 、 tempdb 、 model 4 个数据库，这 4 个数据库称为系统数据库，用来存储 SQL Server 系统运行时会用到的相关信息。

master

master 数据库是 SQL Server 中最重要的数据库，记录了 SQL Server 系统中的所有系统信息，包括登入账户、系统配置和设置、服务器中数据库的名称、相关信息和这些数据库文件的位置，以及 SQL Server 初始化信息等。

tempdb

tempdb 数据库是存在于 SQL Server 会话期间的一个临时性的数据库。一旦关闭 SQL Server ，tempdb 数据库保存的内容将自动消失。重新启动 SQL Server 时，系统将重新创建新的，且内容为空的 tempdb 数据库。

tempdb 数据保存的内容主要包括：显示创建临时对象，例如表、存储过程、表变量或游标；所有版本的更新记录； SQL Server 创建的内部工作表；创建或重新生成索引时，临时排序的结果。

model

model 系统数据库是一个模板数据库，可用作建立数据库的模板。它包含了建立新数据库时，所需的基本对象，如系统表、登录信息等。 model 系统数据库是 tempdb 数据库的基础，由于每次启动 SQL Server 时都会创建 tempdb 。因此 model 数据库必须始终存在于 SQL Server 系统中，用户不能删除该系统数据库。

model 数据库中的所有用户定义对象都会复制到所有新创建的数据库中。可以向 model 数据库中添加任何对象（例如表、视图、存储过程、数据类型等），以将这些对象包括到所有新建数据库中。

msdb

msdb 系统数据库用于提供 SQL Server 代理服务，如果不使用代理服务，就不会使用到该系统数据库。

SQL Server 代理服务是 SQL Server 中的一个 Windows 服务，作用是代理执行所有 SQL 的自动化任务，以及数据库事务性复制等无人值守任务。

用户数据库

用户数据库是指除系统数据库外，用户根据应用的需求自行创建的数据库。

基础知识

  1. SQL Server数据库属于哪一种数据库：关系数据库
  2. SQL Server数据库是由哪家公司创建的：Microsoft
  3. tempdb数据库的作用：存储临时对象
  4. 是否可以在tempdb中创建应用系统所需的用户表：不可以，因为系统重启后tempdb数据库的数据会被清空
  5. 如果有多个数据库需要创建，他们都有若干相同的数据表和用户定义数据类型，下面哪些做法是可行的：（1）在系统数据库model中创建这些相同的数据表和用户定义数据类型（2）在第一个用户数据库建成后，其他用户数据库复制这些表和用户定义数据类型（3）在每一个数据库中重复这些基本表和用户定义数据类型的创建操作（4）以上几种方法都可以

复杂查询

基础知识

  1. 关于自身连接，下面说法正确的有：（1）自身连接的表必须要取别名（2）自身连接的目标列表达式必须带上表的别名（3）自身连接往往用于查询同一个表中不同属性之间满足某种比较条件的结果
  2. 关于自然连接，下面说法正确的有：（1）自然连接可以理解为是主外码值相等且目标列表达式去掉外码保留主码的连接（2）两个表即便没有参照完整性也可以有自然连接（3）自然连接的两个表中必须要有两组相同的属性名及其数据类型和宽度
  3. 关于嵌套查询，下面说法正确的有：（1）嵌套查询就是查询的条件表达式中含有查询块的查询（2）不相关子查询的执行顺序从层查询开始逐级向上一层推进，查询的结果放在上一层查询的条件表达式的常量部分（3）相关子查询一定有WHERE子句，而且都会带有父查询的某个属性作为条件表达式中常量部分的内容（4）某些嵌套子查询也可以用连接查询来实现
  4. 关于集合查询，下面说法正确的有：（1）集合查询就是把表的查询操作当集合操作理解的查询（2）集合查询要求参与运算的子查询的目标列表达式必须个数相同且能按对应顺序进行比较（3）有些集合查询也可以用嵌套查询或连接查询替代（4）集合查询的查询表目标列表达式为第一个子查询的目标列表达式
  5. 关于基于派生表的查询，下面说法正确的有：（1）派生表在SELECT语句中必须使用别名（2）派生表的目标列表达式只有在含有聚集函数、计算表达式等情况下才必须使用别名

SQL的数据更新功能

基础知识

  1. 如果需要增加一个或者多个数据行，SQL的命令动词是：INSERT
  2. 对属性有默认值的数据插入，下面说法正确的是：如果插入数据时没有指定该属性值，在设置默认值之后才会插入默认值
  3. 如果需要删除某个属性值，SQL的命令动词是：UPDATE

  10. 创建视图时，可以通过隐藏基本表中的敏感信息起到一定的数据泄露的保护作用。（√）

数据库的完整性控制

基础知识

  1. 数据库的三类完整性为：实体完整性、参照完整性、用户定义完整性
  2. 实体完整性规则是：主属性不能为空
  3. 下面哪些操作时实体完整性控制需要检查的操作：（1）插入新的元组（2）修改码值
  4. 若R和S分别为具有参照完整性的两张表，其中，R是主表，S是从表，则下面表述正确的有：（1）S中的外码的取值要么引用了R中的码值，要么为空值（2）R中必须包含S中外码所对应的码（3）删除R或修改R被S引用的码值可能破坏参照完整性（4）修改S的外码值或插入新的元组可能破坏参照完整性
  5. 判断该说法是否正确：参照完整性规则就是对外码取值的约束条件。（√）
  6. 判断该说法是否正确：实体完整性规则就是对码值的取值约束条件。（√）
  7. 判断该说法是否正确：用户定义完整性规则就是实际应用系统对基本表中单个属性的取值或者多个属性间取值相关性的约束条件。（√）
  8. 实体完整性控制中，违约检查的内容有：（1）插入新的元组（2）修改已有元组，且修改的属性涉及到主属性
  9. 为了保证数据库的完整性，DBMS必须提供的功能应该包括：完整性定义、违约检查、违约处理
  10. 若参照完整性受到破坏，DBMS采取的违约处理措施包括：拒绝执行、级联执行、当外码是非主属性时，将外码值置为空

存储过程的创建与调用

基础知识

  1. 关于存储过程，下面说法正确的有：（1）存储过程是为了完成某种特定功能而编写的一段在服务器上执行的SQL 语句和流程控制语句的集合（2）存储过程在服务器端对数据库记录进行处理，再把结果返回到客户端（3）SQL Server 提供了用户自定义存储过程和系统存储过程，用户定义存储过程可根据用户的需求自行创建、管理和使用，系统存储过程可以直接调用（4）存储可以被多次调用
  2. 下面对存储过程的作用及其原因描述正确的有：（1）存储过程可以把相应功能执行过程中所涉及的数据结构和处理逻辑封装起来，故可被作为一种安全机制来充分利用（2）存储过程可以当作参数化的视图，可作为一种外模式/模式映像来实现应用程序与数据库间的逻辑独立性（3）存储过程的处理在服务器端完成后才把最终结果返回给客户端，处理过程中不需要向客户端返回中间结果，是减少网络流量的一种有效手段（4）如果存储过程的处理逻辑有改变，在参数不变的情况下就只需要修改过程体，因此带来了非常好的可维护性
  3. 关于游标，下面说法正确的有：（1）游标是一个查询结果集的缓冲区，用于平滑一次一行元组的数据处理方式和数据库一次一集合的数据处理方式（2）如果不是一次对整个SQL结果集进行同一种操作，可考虑使用游标（3）可以基于游标位置对表中数据进行删除或更新（4）可把游标理解为指针，用户对游标的处理就是对游标指针指定位置的数据进行处理
  4. 判断该说法是否正确：声明游标相当于定义一个变量，只有打开游标才能将查询结果放到缓冲区。（√）
  5. 关于FETCH命令，下面说法正确的有：（1）在每次用 FETCH 命令从游标中读取数据时都应检查 @@FETCH_STATUS 全局变量，以确定上次 FETCH 操作是否成功（2）只有在 FETCH 命令被成功执行的情况下，才能对获取的行进行针对性处理（3）FETCH 命令用于从游标中读取行并选择性处理（4）FETCH 命令是每个游标处理过程中都必须要使用的命令
  6. 关于存储过程的参数，下面说法正确的有：（1）可以没有参数（2）可以带一个或多个输入参数和输出参数（3）可以只有输入参数，且输入参数可以带默认值（4）可以只有输出参数
  7. 关于 SQL Server 局部变量，下面说法正确的有：（1）须用 @ 开头（2）必须先定义后使用
  8. 关于 SQL Server 的循环结构，下面说法正确的有：（1）可以使用 BREAK 命令跳出循环体（2）可以使用 CONTINUE 命令跳转到下一个满足条件的循环
  9. 关于 SQL Server 脚本的注释符，下面说法正确的有：（1）使用“–”作为单行注释符，后面无需加上空格（2）使用“/…/”作为多行注释符
  10. 关于 CASE 多分支语句，下面说法正确的有：（1）CASE 和 END 必须配对使用且各占一行（2）若CASE 后面无内容，则 WHEN 后面必须是条件表达式（3）若 CASE 后面有内容，则 WHEN 后面必须是值或能有准确结果的计算表达式（4）对于属性值根据不同情况进行不同更新的应用中，若使用 CASE 分支的方法，且只有多个 WHEN…THEN…而没有ELSE，则不满足前面多个分支条件的值会被置为空
  11. 关于存储过程的输入参数，下面说法正确的有：（1）因为是局部变量，所以输入参数标识符必须带上‘@’且需要声明数据类型和宽度，可以带上默认值（2）在过程体中可以当作常量使用
  12. 关于输出参数，下面说法正确的有：（1）可用于把存储过程执行的结果带出过程体后在应用程序中继续使用（2）调用时必须带上’OUT’或’OUTPUT’，否则无法将值带出过程体（3）其作用类似于函数中的形参

DML触发器

什么是触发器？

在 SQL 中，触发器（Trigger） 是一种特殊的存储过程，它会在特定的数据库事件发生时 自动执行，无需手动调用。

什么是DML触发器？

DML 触发器（Data Manipulation Language Trigger） 是一种在执行 INSERT、UPDATE 或 DELETE 操作时，自动执行的数据库触发器。
