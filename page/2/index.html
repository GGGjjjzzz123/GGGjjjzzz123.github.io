优化算法

优化算法的理解（数学角度）

优化算法是用来解决优化问题的方法，优化问题是优化算法研究的对象和应用领域、

不同的优化问题用不同的优化算法来解决

什么是优化问题：对于某一问题，有求出最优解的需要，为优化问题

优化问题的类型：线性、非线性、约束、不约束、凸优化、非凸优化等

:::success
理解

优化问题定义了目标函数、决策变量、约束条件等要素，而优化算法的任务就是在给定的这些条件下，寻找最优解。优化算法通过对决策变量进行调整，探索可行解空间，最终找到能够使目标函数达到最优（最大或最小）的解。

  • 优化问题的例子：例如，最大化一个工厂的生产利润，或最小化一个交通网络的运输成本。
  • 优化算法的例子：为了解决这个问题，可能使用梯度下降法、遗传算法、模拟退火算法等。

:::

优化算法的发展脉络

:::success
理解

在很久很久以前，人们想解决一些最优化的问题，比如：

- 旅行商问题（TSP）：商人如何规划路线，让自己走的路最短，但还能到达所有城市？
- 资源分配问题：如何分配有限的资源，让收益最大化？

当时，还没有计算机，一切都靠数学家手动推导公式，找出最优解。

:::

1. 早期发展 (19世纪末-20世纪初) ：数学家时代

特点：在人类尚未拥有计算机的年代，数学家们通过手工计算和理论推导来解决优化问题。“解析方法”

  • 变分法（Calculus of Variations）：这是一种在18世纪末到19世纪初提出的方法，用于处理最优化问题，尤其是求解最小化问题。变分法是最早的一类优化方法之一，主要用于求解物理学中的最短路径、最小作用量等问题。
  • 最小二乘法（Least Squares）：由高斯（Carl Friedrich Gauss）和勒让德（Adrien-Marie Legendre）在19世纪初独立提出，最小二乘法成为了数据拟合和回归分析中的基本方法，广泛应用于统计学和工程领域。

2. 线性规划和基本方法（1940s-1950s）：计算机初登场

20世纪中期，工业革命与计算时代，计算机的出现使得优化问题的求解进入了一个新时代，人们开始用数值方法求解大规模优化问题。

  • 线性规划的起点：优化算法的历史可以追溯到1947年，美国数学家乔治·丹茨格提出了**线性规划**问题，即在一组线性约束下，找到使目标函数最大或最小的变量值。这是最早期的优化问题之一。

:::success
理解：

假设你是一个工厂的经理，想最大化生产利润，而生产过程有材料和时间的限制。你可以使用线性规划来计算最佳的生产策略。

线性规划的目标函数为线性（一次函数），变量的约束条件也是线性

:::

  • 线性规划的求解方法：
    • 单纯形法（Simplex Method）（沿着边界搜索）1947：丹茨格提出了单纯形法。这是一种迭代算法，通过不断在可行解的边界上移动来寻找最优解。尽管在理论上可能有些问题，但在实际应用中单纯形法非常高效，广泛用于实际问题中。

:::success
理解

单纯形法不是遍历方法，它是一种基于目标函数改善的迭代算法，每次迭代都会优化目标函数，而不需要检查所有解,只会迭代更优的解。

:::

- **内点法（Interior-Point Method）（在可行域内部搜索）**：** **<u>最早的理论可追溯到 1950年代，但真正广泛应用于计算的是 1984年，由 纳伦德拉·卡马尔卡（Narendra Karmarkar） 提出的 Karmarkar算法，极大地推动了线性规划求解的发展。</u>**  **通过从可行域的内部出发，沿着曲线逼近最优解。内点法在大规模线性规划问题中表现更为高效。
  • 约束优化问题
    • 很多实际问题不仅仅需要优化目标函数，还涉及到约束条件，例如资源限制、物理限制等。拉格朗日乘数法（Lagrange Multiplier）和KKT条件（Karush-Kuhn-Tucker）被提出，用来解决带约束的优化问题，成为了优化领域的重要工具。

3. 引入非线性和约束优化（1960s-1970s）： 数值优化的崛起

计算机的兴起，随着优化问题越来越复杂，研究者们开始研究非线性优化方法。

  • 非线性优化（Nonlinear Optimization）：进入1960年代，很多实际问题的目标函数不再是简单的线性函数，开始出现**非线性优化问题**。随着对更复杂优化问题的需求，非线性优化逐渐成为研究的热点。各种非线性优化方法，如牛顿法、梯度下降法、拟牛顿法等相继引入。

:::success
理解

非线性优化指的是目标函数或约束条件中至少有一个是非线性的优化问题。

:::

- **非线性优化方法主要分为两大类**

解析方法（基于数学推导）

        + 适用于某些特定的非线性优化问题，如凸优化问题（无局部最优）。
        + 包括梯度法、牛顿法等。

数值优化方法（基于数值计算）

        + 适用于大多数复杂的非线性优化问题。
        + 包括一阶方法（梯度下降）、二阶方法（牛顿法）、启发式算法（遗传算法、粒子群优化等）。

:::success
梯度法（Gradient Methods）

梯度法是一类利用梯度信息进行优化的算法的统称，既包括梯度下降法，也包括梯度上升法。

梯度法的核心思想：

- **梯度下降（Gradient Descent, GD）**：沿着负梯度方向移动，以最小化目标函数。
- **梯度上升（Gradient Ascent）**：沿着正梯度方向移动，以最大化目标函数。

牛顿法的理解

假设你站在一座山的某个位置，目标是找到山的最高点（或最低点）。牛顿法就像是你根据山的坡度（导数）和曲率（二阶导数）来选择下一步的方向和步伐——不仅看坡度的大小（即一阶导数），还考虑坡度变化的快慢（即二阶导数），从而更准确地选择如何走到最顶点或最底点。

:::

  • 启发式优化方法：随着计算能力的提升，启发式方法在求解复杂的组合优化问题中得到了应用。
    • 遗传算法（Genetic Algorithm）：模拟生物进化的自然选择过程，通过选择、交叉、变异等操作优化解。

4. 启发式算法的兴起（1980s-1990s）：向自然界学习——人工智能的启发

计算机越来越强，研究者们开始观察自然界是如何优化问题的，于是他们模仿生物的进化、蚂蚁的觅食、金属的退火过程，发明了各种启发式优化算法。

  • 启发式算法：随着计算机科学的发展，许多复杂的优化问题（尤其是组合优化问题）变得难以用精确的数学方法解决。这时，启发式算法的出现为我们提供了新的思路。这些算法通过模拟自然界的现象来寻找“好”的解决方案，虽然它们不能保证找到最优解，但通常能够找到一个足够好的解。
    • 模拟退火算法（Simulated Annealing）：受到金属退火过程的启发，这种算法通过模拟“降温”过程，逐步寻找最优解。
    • 粒子群算法（Particle Swarm Optimization）：模拟鸟群觅食行为，每个解（粒子）在搜索空间中移动，受自身经验和群体最优经验的影响。

:::success
此时，1984年，印度数学家卡迈克尔提出了内点法，这是一种全新的线性规划求解方法，突破了单纯形法的限制。内点法被认为是大规模线性规划的一个重要进展。

:::

5. 现代阶段（2000s至今）：AI与深度学习的优化挑战

随着人工智能的发展，深度神经网络的训练成为一个重要的优化问题。优化算法也迎来了新的变革，特别是在深度学习和强化学习领域。

  • 深度学习中的优化：随着人工智能（AI）和深度学习的迅猛发展，优化问题的研究逐渐集中在如何高效训练深度神经网络。为了解决训练中的问题（如收敛速度慢、容易陷入局部最优解等），研究者提出了更先进的优化算法，如随机梯度下降（SGD）和Adam算法，这些算法大大加速了训练过程并提高了性能。

:::success
- 随机梯度下降（SGD）：主要用于机器学习和深度学习中最小化损失函数，通过每次使用一个小批量数据（mini-batch）来计算梯度，避免了全数据训练带来的计算开销，加快训练速度。
- Adam算法： 用来训练神经网络。可以把它看作是 SGD的加强版，引入了动量和自适应学习率，使得优化过程更加稳定和高效。

:::

:::success
深度学习优化

就是指如何调整神经网络的参数（权重和偏置）来尽可能降低损失函数的过程。也就是说，通过某种“优化算法”让模型的预测尽可能准确。

:::

  • 分布式优化与大数据：随着大数据和云计算的兴起，越来越多的优化问题需要在大规模数据集上进行求解。为了应对这种需求，分布式优化应运而生。通过将计算任务分布到多个计算节点，可以更快地处理海量数据，如分布式梯度下降（Distributed SGD）。

优化算法从最早的线性规划到今天应用于深度学习、量子计算等前沿领域（优化算法的发展从数学家手工推导，到计算机辅助求解，再到人工智能和量子计算的结合），经历了从简单的数学模型到复杂的启发式方法，再到现代智能算法的演变。每个阶段的进步都离不开对实际问题的需求，以及计算能力的提升。从最早的求解数学问题到现在处理复杂大规模数据和智能系统的优化，优化算法一直在推动科学和技术的进步。

2.1949年 Hebb学习规则

  • Hebb规则（无监督学习规则）：心理学家赫布（Hebb）出版了《The Organization of Behavior》（行为组织学），他在书中提出了突触连接强度可变的假设。可变性是学习和记忆的基础。

此时，认为人脑神经细胞的突触（也就是连接）上的强度是可以变化的，科学家们开始考虑用调整权值的方法来让机器学习

  • Delta学习规则（有监督学习规则）1960年：Delta学习规则（Delta Rule）由伯纳德·维德罗（Bernard Widrow）和马塞尔·霍夫（Marcian Hoff）于1960年在开发自适应线性元件（ADALINE）时提出。该规则是一种梯度下降学习算法，旨在通过最小化输出与期望值之间的均方误差来更新单层神经网络中输入权重。

3.1957年 两层神经元组成的神经网络——感知器

  • 感知器模型：1957 年，罗森勃拉特（Rosenblatt）以M-P 模型为基础，提出了感知器（Perceptron）模型，由两层神经元组成的神经网络，即为单层的人工神经网络，以区别于较复杂的多层感知机（Multilayer Perceptron）。一个一个样本进行调节。激活函数实际为阶跃函数。

:::success

4.伊辛模型（Ising Model）1920年

  • 伊辛模型（Ising Model）是统计物理学中用于描述铁磁性物质相变行为的数学模型。该模型由物理学家威廉·楞次（Wilhelm Lenz）于1920年提出，并由其学生恩斯特·伊辛（Ernst Ising）在1924年的博士论文中研究。
  • 模型概述：伊辛模型将物质视为由规则排列的自旋组成的晶格系统。每个自旋（磁矩）具有两个可能的取向，通常表示为+1（向上）或−1（向下），对应于自旋的两种状态。这些自旋位于晶格的节点上，并且仅与其相邻的自旋发生相互作用。相邻的自旋如果一致，则具有比不一致的自旋更低的能量；系统趋向于最低能量，但热量会干扰这种趋势，从而产生不同结构相的可能性。该模型允许识别相变，作为现实简化的模型。二维正方形晶格伊辛模型是显示相变的简单统计模型之一。
  • 相变行为：
    • 伊辛模型成功地解释了铁磁物质在临界温度（居里温度）下的相变现象：
      • **低温阶段（T < T_c）：**自旋倾向于自发地对齐，导致系统表现出宏观磁化，即铁磁性。
      • 高温阶段（T > T_c）：热扰动使自旋随机取向，系统呈现无序状态，宏观上无磁化，即顺磁性。
    • 在临界温度T_c附近，系统经历从有序到无序的相变，表现出临界现象，如自发磁化的消失和磁化率的发散。

5.1982年 Hopfield神经网络模型

  • Hopfield模型：1982年，受到伊辛模型的启发，神经网络是否可以全连接。美国物理学家霍普菲尔德（Hopfield）提出了一种离散神经网络，即离散Hopfield网络，从而有力地推动了神经网络的研究。在网络中，Hopfield把最优化问题的目标函数转换成 Hopfield 神经网络的能量函数，通过网络能量函数最小化来寻找对应问题的最优解,证明了网络的稳定性。

（Hopfield神经网络是一种递归神经网络，从输出到输入有反馈连接—>全连接）

  • 1984年，Hopfield 又提出了一种连续神经网络，将网络中神经元的激活函数由离散型改为连续型。
  • 1985 年，Hopfield和Tank利用Hopfield神经网络解决了著名的旅行推销商问题（TSP）
  • 功能：有联想记忆的功能，输入图片的一部分，可以自动补全

:::success
理解

对于Hopfield网络，把网络的基态训练好，随便输入图形就会回到网络基态（对应的网络能量函数的最小值）。原理是：该网络的工作原理基于能量函数，系统通过状态更新逐步减少能量，最终收敛到一个稳定状态，对应于存储的模式。

:::

:::color1
提出的原因

  • 霍普菲尔德原本是物理学家，研究磁性材料中的自旋系统（伊辛模型）。他发现，磁性材料中的原子自旋会自发排列以降低系统能量，这与神经网络的状态演化有相似之处
  • 他意识到，神经网络的动力学行为可以用“能量函数”来描述，类似于物理系统的自由能。通过引入能量最小化的概念，他构建了一个能够自动收敛到稳定状态的网络模型
  • 这一思想使得霍普菲尔德网络能够像物理系统一样，从初始状态演化到能量最低的稳定状态，从而实现模式存储与联想记忆的功能

:::

6.1983年 “隐单元”的概念被首次提出 —> 基于此设计出了玻尔兹曼机

  • 1985年，杰弗里·辛顿（Geoffrey Hinton）和泰伦斯·塞诺夫斯基（Terry Sejnowski）提出了玻尔兹曼机，这是一种随机递归神经网络，能够学习数据的概率分布。玻尔兹曼机通过引入随机性和模拟退火过程，使网络能够跳出局部最小值，找到全局最优解。该模型的理论基础源于统计力学中的玻尔兹曼分布，体现了物理学与人工智能的交叉融合。

:::success
理解

玻尔兹曼机，在Hopfield网络上增加一些隐藏单元，是一种由随机神经元全连接组成的反馈神经网路，其中只包含一个可见层和一个隐藏层。

:::

  • 玻尔兹曼机和Hopfield网络的区别：
    • H网络的神经元要么是1，要么是0，玻尔兹曼机的神经元是在两个状态之间来回横跳，是一种概率状态，满足玻尔兹曼分布。同时，也受温度的影响，可以在温度的帮助下找到全局最优解（模拟退火过程）
  • 受限玻尔兹曼机：全连接的玻尔兹曼机参数过多 ，训练起来太麻烦，故将可见层的连接断开，只与隐藏层相连。

:::color1
提出的原因

受霍普菲尔德网络（Hopfield Network）的启发，研究者希望构建一种能够模拟人脑联想记忆功能的网络模型。然而，霍普菲尔德网络在处理复杂模式时存在容量限制和易陷入局部最优的问题。玻尔兹曼机采用模拟退火算法，通过逐步降低系统的“温度”，使网络状态逐渐趋于稳定，从而找到全局最优解。

在1980年代，传统的神经网络（如感知机）主要依赖监督学习，需要大量标注数据，且难以处理复杂的概率分布问题。玻尔兹曼机的提出是为了探索一种无监督学习机制，能够从数据中自动学习概率分布，而无需依赖人工标注。

玻尔兹曼机借鉴了统计物理学中的玻尔兹曼分布（Boltzmann Distribution），将神经网络的状态建模为能量函数的最小化问题。其核心思想是：

  • 每个神经元的状态（激活/未激活）由能量函数决定。（随机性）
  • 网络的稳定状态对应于能量最低的配置，即最可能的概率分布。

:::

  • 不足：一次神经网络的仍然耗时太久，存在局部最优解的问题

  • 扬·勒昆（Yann LeCun）等人将反向传播算法应用于卷积神经网络，成功用于手写数字识别，开发了LeNet模型。

  • 塞普·霍赫赖特（Sepp Hochreiter）和于尔根·施密德胡贝尔（Jürgen Schmidhuber）提出了长短期记忆网络（LSTM），解决了递归神经网络中的长期依赖问题。

10.2006年 深度信念网络

  • 杰弗里·辛顿等人提出了深度信念网络（Deep Belief Network），引发了深度学习的研究热潮。深度信念网络（Deep Belief Network，简称DBN）是一种深度生成式模型，由Geoffrey Hinton于2006年提出。它通过堆叠多个受限玻尔兹曼机（Restricted Boltzmann Machines，RBM）构成，每个RBM的隐藏层作为下一层的可见层，从而形成多层结构。
  • 结构与训练过程（预训练+微调）：
    1. 逐层预训练：首先，对最底层的RBM进行无监督训练，使其学习输入数据的特征表示。训练完成后，固定该层的参数，将其隐藏层的激活值作为下一层RBM的输入。
    2. 重复上述过程：对每一层RBM重复上述训练步骤，直至所有层都经过预训练。
    3. 微调：在完成所有层的预训练后，可以采用有监督的方式（如反向传播算法）对整个网络进行微调，以适应特定的任务需求。

:::success

  • 预训练：可以方便的让神经网络中的权值找到一个接近最优解的值
  • 微调：对整个网络进行优化训练

:::

  • 意义：
    • 大幅度减少训练多层神经网络的时间
    • 更深入的表示特征，更强的函数模拟能力
    • 将多层神经网络相关的学习方法命名为深度学习，
  • 不足：会出现过拟合现象

需要注意的是，随着深度学习技术的不断发展，新的模型架构（如卷积神经网络和循环神经网络）在许多应用中表现出更优异的性能，因此DBN的使用逐渐减少。

神经网络和优化算法的关系：

1. 神经网络本身是一个算法：

神经网络（如人工神经网络）本身是一种解决机器学习任务的算法。它由多个神经元组成，层与层之间通过连接进行信息传递和处理。这种网络需要训练来学习从输入到输出的映射关系。

2. 训练神经网络是一个优化问题：

在训练神经网络时，我们面临一个优化问题：即如何调整网络中的权重和偏置，使得网络的预测结果尽可能接近真实值，通常通过最小化损失函数（如均方误差或交叉熵）来进行优化。

这个优化过程就是一个典型的优化问题，其中我们试图最小化损失函数，所以训练神经网络可以看作是在优化一个目标函数（损失函数）。

3. 训练神经网络依赖优化算法：

训练神经网络时，使用的优化算法（如梯度下降法）用来找到最优的网络参数（权重和偏置）。这些优化算法通过反向传播和梯度计算来调整权重，以减少训练误差。

  • 梯度下降（Gradient Descent）：最常用的优化算法，它通过计算损失函数的梯度，并根据梯度方向更新网络权重，目的是最小化损失函数。
  • Adam、RMSProp 等优化算法则是对梯度下降法的改进，能在训练时更有效地调整学习率。

蒙特卡洛方法

只要加入了随机性，就可以说它是蒙特卡洛方法

  • 概念：蒙特卡罗法也称统计模拟法、统计试验法。是1940年代中期提出的一种把概率现象作为研究对象的数值模拟方法。是按抽样调查法求取统计值来推定未知特性量的计算方法。蒙特卡罗是摩纳哥的著名赌城，该法为表明其随机抽样的本质而命名。故适用于对离散系统进行计算仿真试验。在计算仿真中，通过构造一个和系统性能相近似的概率模型，并在数字计算机上进行随机试验，可以模拟系统的随机特性。

注意：这个方法是一类基于概率的方法的统称，不是特指一种方法。

  • 蒙特卡罗解决的两类问题：
    1. 模拟预测问题：通过将含有随机性无法直接按数值进行求解的问题进行建模，然后进行多次模拟，最后的输出就是对实际发生结果的预测。
      • 例如在核物理研究中，分析中子在反应堆中的传输过程。中子与原子核作用受到量子力学规律的制约，人们只能知道它们相互作用发生的概率，却无法准确获得中子与原子核作用时的位置以及裂变产生的新中子的行进速率和方向。科学家依据其概率进行随机抽样得到裂变位置、速度和方向，这样模拟大量中子的行为后，经过统计就能获得中子传输的范围，作为反应堆设计的依据。
    2. 复杂多维积分问题：所求解问题可以转化为某种随机分布的特征数，比如随机事件出现的概率，或者随机变量的期望值。通过随机抽样的方法，以随机事件出现的频率估计其概率，或者以抽样的数字特征估算随机变量的数字特征，并将其作为问题的解。这种方法多用于求解复杂的多维积分问题。
  • 经典例子：
    • 计算不规则图形的面积
    • 求复杂函数的定积分
    • 圆周率π 值求解
  • 常见的蒙特卡洛法：
    • 直接抽样（可以对目标分布直接进行采样）：
      • 适用于可以直接从目标分布中采样的情况。例如，从标准正态分布中生成随机数。
    • 接受-拒绝抽样法（目标分布复杂，无法直接进行采样）：
      • 适用于目标分布复杂，无法直接采样的情况。该方法通过从一个易于采样的建议分布中生成样本，并根据一定的接受概率决定是否接受该样本。
    • 重要性抽样法（目标分布复杂，无法直接进行采样）：
      • 适用于目标分布复杂，但可以从另一个易于采样的分布中生成样本，并通过对样本赋予权重来估计目标分布的特征。

:::success
理解

核心就是通过大量随机试验（随机抽样）来逼近问题的真实解。

蒙特卡罗法要解决的问题是，假设概率分布的定义己知，通过抽样获得概率分布的随机样本，并通过得到的随机样本对概率分布的特征进行分析。所以蒙特卡罗法的核心是随机抽样

马尔科夫链

  • 概念：俄国数学家 Andrey Andreyevich Markov 研究并提出一个用数学方法就能解释自然变化的一般规律模型，被命名为马尔科夫链（Markov Chain）。马尔科夫链为状态空间中经过从一个状态到另一个状态的转换的随机过程，该过程要求具备“无记忆性 ”，即下一状态的概率分布只能由当前状态决定，在时间序列中它前面的事件均与之无关。这种特定类型的“无记忆性 ”称作马尔可夫性质。

:::success
马尔科夫链的思想：过去所有的信息都已经被保存到了现在的状态，基于现在就可以预测未来。

:::

  • 马尔科夫链在机器学习中的应用：语言模型N-Gram是一种简单有效的语言模型，基于独立输入假设：第 n 个词的出现只与前面 N-1 个词相关，而与其它任何词都不相关 。整句出现的概率就是各个词出现概率的乘积。这些概率可以通过直接从语料中统计 N 个词同时出现的次数得到。

稳态：

稳态不受初始状态影响，但并不是所有的马尔科夫链都具有唯一的稳态分布

马尔科夫蒙特卡洛（MCMC）方法

  • 概念：
    • 马尔可夫链蒙特卡罗法也是一种随机采样方法，被应用于概率分布的估计、定积分的近似计算、最优化问题的近似求解等问题，特别是被应用于统计学习中概率模型的学习与推理，是重要的统计学习计算方法。
    • 适合于概率密度函数复杂，不能直接抽样的情况，相比传统的蒙特卡罗法，如接受-拒绝法、重要性抽样法，马尔可夫链蒙特卡罗法更适合于随机变量是多元的、密度函数是非标准形式的、随机变量各分量不独立等情况，且抽样效率更高。
  • 基本思想：

可见，若随机变量 x 是离散的，则应定义相应的离散状态马尔可夫链；若随机变量 x 是连续的，则应定义相应的连续状态马尔可夫链

:::success
理解

蒙特卡罗法是通过从概率模型的随机抽样进行近似数值计算的方法。

马尔科夫链蒙特卡洛法是构建马尔科夫链为概率模型，然后再基于马尔科夫链进行蒙特卡洛法进行随机采样，由于马尔可夫链的稳态，可进行近似数值的计算。

:::

如何构建具体的马尔科夫链成为马尔科夫链蒙特卡洛法的关键。因此，马尔科夫链蒙特卡洛法中的具体方法（Metropolis 算法 、Metropolis-Hastings 算法）的核心为构建一个的马尔科夫链使其平稳分布为目标分布

Metropolis 算法

  • 概念：Metropolis 算法是 1953 年由 Nicholas Metropolis 等人提出的 MCMC 方法，旨在从复杂的概率分布中采样。其核心思想是构建一个马尔可夫链，使其平稳分布为目标分布。该算法使用对称的建议分布（即从状态 A 到 B 的概率等于从 B 到 A 的概率），并通过接受-拒绝机制决定是否接受新状态。
  • 这个算法的核心接受规则就是Metropolis 准则。

Metropolis-Hastings 算法

  • 概念：Metropolis-Hastings 算法是最基本的马尔可夫链蒙特卡罗法， Metropolis 等人在 1953 年提出原始的算法，Hastings 在1970 年对之加以推广，形成了现在的形式。Metropolis-Hastings 算法是对 Metropolis 算法的推广，由 W.K. Hastings 于 1970 年提出。它放宽了对称建议分布的限制，允许使用非对称的建议分布。为此，引入了一个更一般的接受概率公式，以确保马尔可夫链的平稳分布仍为目标分布。

Metropolis抽样

  • 概念：Metropolis 抽样通常是指使用 Metropolis 或 Metropolis-Hastings 算法从目标分布中生成样本的过程。这个术语强调了通过这些算法进行的实际采样操作。

Metropolis 准则

  • 概念及由来：
    • Metropolis 准则最初是为了解决物理系统中粒子的热平衡模拟问题。在模拟过程中，需要从玻尔兹曼分布中采样粒子的状态，但直接采样困难（概率密度函数复杂）。Metropolis 等人提出了通过构建马尔可夫链，并使用上述接受准则，从而实现对目标分布的有效采样 。
    • 1953 年，Metropolis 提出重要性采样，即以概率来接受新状态，而不是使用完全确定的规则，称为 Metropolis 准则，可以显著减小计算量。
    • Metropolis 准则是 Metropolis 算法中的核心机制，用于决定是否接受一个新的候选样本。该准则的提出源于 1953 年 Nicholas Metropolis 等人发表的论文《Equation of State Calculations by Fast Computing Machines》，旨在通过构建马尔可夫链，从复杂的概率分布中采样，以模拟物理系统的热力学性质 。
  • Metropolis 准则在模拟退火算法中的应用

:::success
接受-拒绝抽样法和 Metropolis准则的区别

接受-拒绝抽样法适用于低维、简单分布的直接采样，采样的样本之间是独立的。

Metropolis 准则是 MCMC 方法中的核心，用于从高维复杂分布中采样，采样样本之间不是相互独立的 （样本构成马尔可夫链）

:::

:::success
蒙特卡洛法和马尔科夫链蒙特卡洛法的区别

蒙特卡罗法中的抽样样本是独立的，而马尔科夫链蒙特卡罗法中的抽样样本不是独立的，样本序列形成马尔科夫链

马尔可夫链蒙特卡罗法中得到的样本序列，相邻的样本点是相关的，而不是独立的。因此，在需要独立样本时，可以在该样本序列中再次进行随机抽样，比如每隔一段时间取一次样本，将这样得到的子样本集合作为独立样本集合

:::

:::success
模拟退火算法和Metropolis算法之间的关系

两者都使用 Metropolis 接受准则来决定是否接受新状态。

模拟退火算法可以看作是在 Metropolis 算法的基础上，引入温度参数并设计冷却策略，以实现从探索到收敛的过程。

:::

第一阶段：

由此产生了迫切的实践需求：需要建立一种在对象先验认知较少时仍能普遍适用的优化方法。这种通用方法就是搜索，它能提供寻找最优解所需的信息。

搜索最初是作为求解复杂分析问题时迭代过程的改良而发展起来的。迭代方法的开发催生了搜索理论。搜索程序的硬件实现则促成了各类极值调节器与优化器的诞生，同时也推动了对搜索优化过程的理论研究。极值控制由此应运而生，成为技术控制论的分支领域——它既与迭代过程理论相关联，又与自动控制理论相衔接，同时还与极值实验设计理论紧密相连。

A Note on Research Methodology and Benchmarking Optimization Algorithms

研究发现，虽然有许多函数涵盖了广泛的特征，但有些特征代表性不足，甚至完全没有覆盖。此外，还发现常见的基准测试套件由根据测量特征高度相似的函数组成。

事实上，并不存在标准的测试函数列表。不同的研究人员选择不同的基准测试函数集合，而这一选择并没有任何合理的理由。此外，测试平台的函数数量也差异很大，从两个函数到二十多个函数不等（数量不等）。测试函数的范围包括不同的特征；例如，单峰和多峰、规则和不规则、可分离和不可分离等（特征类型不等）。

多年来，研究人员将多种动态测试问题应用于比较 EA 在动态环境中的性能，例如 Branke 提出的“移动峰值”基准测试（MPB）[1]、Morrison 和 De Jong 引入的 DF1 生成器[7]、Jin 和 Sendhoff 通过动态组合现有静态多目标基准问题的不同目标函数生成的单目标和多目标动态测试问题生成器[2]、Yang 和 Yao 的异或（XOR）算子[11, 12, 13]、Kang 的动态旅行商问题（DTSP）[3]和动态多背包问题（DKP）等。尽管文献中存在多种动态优化问题生成器，但目前还没有一种统一的方法来构建跨越二进制空间、实数空间和组合空间的动态问题。本报告采用文献[4]中提出的广义动态基准生成器（GDBG），该生成器为这三种解空间构建动态环境。特别是在实数空间中，我们引入了一种旋转方法，而不是像 MPB 和 DF1 生成器那样通过移动峰值位置。旋转方法可以克服 MPB 生成器算法在峰值位置从景观边界反弹时每次变化所面临的挑战不均等问题。

优化算法

3：优化算法可分为两种类型：精确（传统）算法和进化算法。精确优化技术使用连续搜索策略，它基于目标函数的导数，即使对于一个小问题，传统优化方法也需要很长时间才能找到精确的解。

确定性算法：最速下降法、牛顿法和拟牛顿法等。精确优化技术使用连续搜索策略，它基于目标函数的导数，即使对于一个小问题，传统优化方法也需要很长时间才能找到精确的解。

进化算法：需要较少的参数。它们不需要连续的目标函数。实际工程问题通常很复杂，它们依赖于许多变量和选择一个好的起始点。因此，许多基于群体的元启发式算法被开发出来以解决复杂问题

4：传统优化方法，如梯度下降法、牛顿-拉夫森法等，虽然强大，但存在显著缺点。首先，它们的作用范围是局部的，在复杂、非凸的景观中往往无法找到高质量的解决方案。其次，这些算法在整个搜索过程中依赖于梯度信息，要求设计空间是连续且光滑的，而现实世界往往并非如此[1]。此外，由于需要频繁进行梯度评估，它们可能计算成本高昂，特别是在高维空间中。当设计空间是离散时，研究人员采用了枚举算法；然而，随着设计变量数量的增加，这些算法的性能会迅速恶化，因为它们受到维数灾难的影响[2]。元启发式算法的使用，如遗传算法（GA）[3]，[4]、粒子群优化[5]蚁群优化算法 [6] 等方法已成为解决此类挑战的强大工具。受自然过程的启发，这些算法在合理的时间范围内高效地为各种优化问题找到近似最优解 [7], [8]。这些算法在解决精确算法不切实际的优化问题时展现了巨大潜力。这些方法为多目标问题提供了效率和可扩展性 [9]，并且可以方便地应用于各种问题领域，包括组合优化、离散优化和连续优化 [10]。它们对不完美的目标函数表现出鲁棒性，这些目标函数通常是间断的、不可微分的和随机的 [11]，并且可以在分布式和并行计算基础设施中实现 [12]。然而，这些元启发式算法的挑战在于它们依赖于广泛的超参数调整，这使开发通用优化方法变得复杂。尽管这些算法旨在通用，但它们的性能可能不一致、不可靠且耗时 [13], [14]。 这种不一致性源于需要调整大量超参数以及其过程固有的随机性，导致在任何给定试验中难以获得可预测的结果[15]。

通常，有四类常用的测试问题：(1) 每年提出的用于竞赛的测试套件，例如 IEEE 进化计算大会（CEC）基准测试[28, 40, 48]和黑盒优化基准测试（BBOB）[3]；(2) 已知基准问题，用于有影响力的文章，例如[54]；(3) 可调基准生成器，例如高斯集（MSG）生成器[17]、多模态景观生成器[41]、NK[6]和 NM[37]生成器；最后，(4) 真实世界问题集合[9]。前三类是模拟问题，在理解真实世界数据方面发挥着重要作用。无免费午餐（NFL）定理[53]表明，当所有可能的问题发生概率相等时，没有一种算法比另一种算法更好。NFL 定理还警告我们泛化优化算法性能的风险，例如算法在人为设计的基准测试上表现良好，但在真实世界问题上表现不佳的风险。 然而，由于不同的问题类别并非完全互斥（例如，人类设计的基准测试和现实世界的问题并不完全如此），不同的问题类别之间必然存在一些重叠，因此可以审慎地推断和预期算法的性能。此外，一个人无法研究现实世界中的所有问题，而且总有机会找到其中的一部分，在这些部分上可以宣布一个算法的性能明显优于另一个算法。

  1. 神经网络发展历史

人工神经网络提供了一种方法，以与人脑相同的方式表征合成神经元，从而解决复杂问题。多年来，特别是从上世纪中期开始，对研究大脑机制和结构的兴趣不断增长。这种日益增长的研究兴趣促使人们基于生物学背景，开发出新的计算模型、连接主义系统或人工神经网络，用于解决模式识别、快速信息处理和适应等复杂问题。

在 20 世纪 40 年代初，麦克洛克和皮茨[37]基于神经元模型研究了几个基本组件互连的潜力和能力。后来，其他人如赫布[38]关注神经系统中涉及的适应规律。1958 年，罗森布拉特[39]提出了感知器这一名称，并设计了一种随后引起广泛关注的结构。1960 年，维德罗和他的学生霍夫[40]提出了感知器训练算法的重要推广，即均方误差（LMS）学习过程，也称为 Delta 规则。该学习规则被用作自适应线性元件在 ADALINE（ADAptive LInear Neuron）网络中。随后，明斯基和帕普特[41]对感知器进行了严格的分析，证明了其许多性质，并指出了几个相关模型的局限性。在 20 世纪 70 年代，格罗斯贝格[42]的工作引起了广泛关注。他的工作基于生物学和心理学的证据，提出了非线性动态系统的新颖结构。1982 年，霍普菲尔德[43]将一种特定的非线性动态结构应用于解决优化问题。 它们都在 ANN 的理论方面进行了开创性研究，特别是从 20 世纪 50 年代和 60 年代开始。

1986 年，PDP 研究小组发表了一系列算法和结果[2]。这篇出版物包含一篇题为“通过误差传播学习内部表示”的文章[1]。这篇文章使 BP 训练算法得到认可，尽管该算法在 1974 年已经被描述过[44]。使用通用 Delta 规则的 BP 算法[1,7]，作为监督训练算法的代表，极大地推动了后续研究，并导致了 ANNs 研究与应用的最大成果，尽管与此同时其他 ANN 架构和训练算法也被开发和应用。1982 年，芬兰教授 Teuvo Kohonen[45]首次描述了一种不同的 ANN 架构，该架构使用无监督的 SOM 学习过程进行训练。这种 Kohonen SOM 算法后来在许多研究和实际应用中得到了广泛应用。1988 年，首次使用了径向基函数神经网络[46]，尽管 RBFs 在 1977 年就已经被介绍过[47]。 径向基函数网络也被广泛使用，具有强大的函数逼近能力，并与 MLP 感知器一起对支持向量机的出现产生了影响[32, 33, 48]。在 20 世纪 80 年代末，标准的多层函数网络被证明在 R 的紧子集上是通用逼近器[49, 50]。多层函数网络具有一个包含有限数量隐藏神经元的隐藏层。这些神经元使用任意的激活函数。理论保证多层函数网络，包括 MLP 网络、径向基函数网络，甚至在一定条件下支持向量机[51]，能够处理高度复杂和非线性的问题。

迄今为止，人工神经网络已在许多工业和商业应用中使用，例如过程建模与控制[52]、字符识别[53]、图像识别[54]、信用评估[55]、欺诈检测[56, 57]、保险[58]和股票预测[59]。在后面的章节中，将回顾人工神经网络在农业和生物工程中的应用。

原文网址：https://wires.onlinelibrary.wiley.com/doi/10.1002/wics.70028?af=R

摘要：

优化中的基准测试是评估机器学习算法和元启发式算法的性能、鲁棒性和可扩展性的关键步骤。尽管基准测试设计趋势不断发展，但合成函数仍然是基本压力测试和理论评估的重要工具。在过去的几十年中，已经开发并推导出多种基准测试和测试函数，但很少关注对这些测试函数进行分类及其使用背后的原理。从这一视角出发，本文回顾并分类了在评估优化器和元启发式算法时常用的各种函数。具体而言，我们根据模态、维度、可分离性、平滑度、约束条件和噪声特性对测试函数进行分类，以提供广泛的视角，帮助为各种算法挑战选择合适的基准测试。此外，本综述还详细讨论了文献中最常用的25个函数，并提出了两个新的、高维、动态且具有挑战性的函数，可用于测试新算法。 最后，本综述确定了当前基准测试实践中的空白和未来研究的方向，并提出了最佳实践和指南。

  1. 引言

机器学习（ML）算法通常依赖参数值的迭代调整来优化目标函数，以解决分类误差最小化、统计模型中最大化似然等优化任务（Sra 等人 2012）。这些任务在学术研究和工业应用中至关重要（例如，为大规模图像识别微调神经网络，以及为个性化体验优化推荐系统结构）。类似地，元启发式算法（如遗传算法和粒子群优化）也能处理挑战传统梯度方法或缺乏解析结构的复杂优化问题（Boussaïd 等人 2013）。虽然梯度下降及其变体在某些 ML 场景中占据主导地位，但元启发式算法在具有崎岖适应度景观、多个极值或复杂约束的领域表现优异。因此，高效优化是 ML 和元启发式算法中许多过程的核心（Sun 等人 2020）。

基准函数和测试函数为评估上述方法的有效性和局限性提供了受控环境（杨 2010）。如果没有适当的基准，就难以比较算法、识别优缺点或完善方法论假设（丰塞卡等 2020）。例如，如果一个算法在某一问题类型上收敛迅速，但如果其性能仅在简单基准上衡量，那么它可能不适合其他场景（肖帕特和托马西尼 2018）。因此，代表性测试函数的开发和选择严重影响新型优化策略所报告的成功或失败。更具体地说，在机器学习研究中，优化贯穿算法开发的多个阶段（即参数初始化、模型架构探索），因此，稳健的优化流程对于实现高性能至关重要。 同样地，尽管有时被视为黑盒方法，元启发式算法依赖于彻底的测试来验证它们在不同景观和约束条件下的能力（Omidvar 等人 2021；Omidvar 等人 2022）。

尽管公认需要基准测试，该领域仍存在测试函数使用不规范和不一致的问题（Longjohn 等人 2024；Bartz-Beielstein 等人 2010）。例如，研究人员经常采用一套有限的知名基准，如 Sphere、Rastrigin、Rosenbrock 和 Ackley 函数，而没有系统地考察它们与新提出的方法的相关性（White 等人 2013）。这种对狭窄基准的过度依赖可能强化算法偏见，并可能导致虚高的性能声明（Runarsson 和 Yao2005）。除了本文讨论的传统基准函数外，一个可配置的基准生成器，称为 GNBG，允许按需调整景观特征，从而能够构建用于深度算法分析的定制测试套件（Yazdani、Omidvar 等人 2023）。此外，传统基准函数往往忽视动态或随机条件，但这些特性在现实场景中正变得越来越普遍。 此外，优化研究中的公平性和可重复性取决于公认基准的统一采用。因此，有必要检验各种函数，以研究诸如模态、维度、可分离性、噪声或约束等属性，从而实现全面的算法评估（Beiranvand 等人 2017；Parejo 等人 2012）。

本综述通过系统化分类和分析测试与基准函数的方法，应对上述挑战。鉴于该领域的广阔性，本综述主要关注合成函数（如纯粹为理论或说明目的设计的函数）。合成函数对于基础压力测试仍具有重要价值，因为它们允许精确控制景观特征，如最优位置或多模态程度（Cheng 等人 2017; Okabe 等人 2004; Jamil 等人 2013）。因此，我们的首要目标是根据模态（单模态与多模态）、维度（低维与高维）以及约束的存在与否等特征，系统性地识别和分类函数。我们的次要目标是强调当前基准函数目录中的空白，这些空白源于多个因素，包括缺乏标准化和动态问题。 最后，我们的综述提出了选择测试函数的最佳实践，并确定了未来研究的挑战和方向，以此鼓励机器学习和元启发式算法社区更加谨慎和一致地采用基准测试。

  1. 背景与文献综述

优化基准测试一直致力于在受控但信息丰富的环境下客观比较不同算法（Chinneck 2008）。早期研究强调使用具有明确闭式表达式的手工设计函数作为参考，以测试新方法/算法（Ŝtefek 2011）。这些初步工作建立了诸如在少量公认基准上评估收敛速度和求解精度等实践（Tedford and Martins 2010）。虽然按今天的标准来看很基础，但这些初始步骤确立了基本主题：函数定义的透明性、可重复的实验设置，以及涵盖广泛的问题特征（Alimoradi et al. 2010）。

当开发者意识到基准测试范围过于狭窄可能会影响算法设计并夸大性能时，一个重要的转折点出现了（Brownlee 2007）。为此，开发者开始记录依赖单一问题比较或仅能捕捉到平滑、低维景观的简单测试套件的潜在风险（Sala and Muller 2020）。在 20 世纪 90 年代和 21 世纪初，研究引入了具有尖锐局部最小值、欺骗性全局结构或变量间耦合的复杂函数。在这些设计之外，研究人员还提出了详尽的框架，按维度、模态和可分离性对函数进行分类（Weise et al. 2014）。随着时间的推移，这些分类使得跟踪算法性能变得更加容易。

一项有影响力的研究通过推广通用实验标准来处理基准测试研究的可重复性问题，包括统一的初始化、一致的停止标准和指标（Moreau 等人 2022）。然而，由于算法种类繁多，它们与同一函数的交互方式各不相同，因此全面标准化仍然模糊不清（Kistowski 等人 2015）。对已发表工作的审视指出，即使一个通用的测试框架似乎难以实现，研究人员仍然可以就共享最佳实践达成共识，例如报告达到特定阈值所需的函数评估次数，以及运行多次独立的实验（López-Ibáñez 等人 2021；Sigmund2022）。

到了 21 世纪初中期，出现了专门的基准测试平台，特别是在进化计算竞赛和黑盒优化基准测试中（Martí等人 2025）。这些举措提供了具有已知属性的官方测试函数套件（van Thieu 2024）。一个显著的结果是，算法开发者必须在一个社区认可的套件中展示其能力（Agushaka 等人 2022）。2010 至 2020 年代见证了基准测试研究的兴趣激增，部分原因是人们担心实现细节或硬件架构的差异混淆了报告结果（Skvorc 等人 2019）。这引发了关于在公共平台上托管的标准化库和代码存储库的讨论（Piotrowski 等人 2023；Mohamed 等人 2023；Mejía-de-Dios 和 Mezura-Montes 2022）。 大量论文展示了分发算法代码和可重复的工作流程脚本的优势（Kämpf 等人 2010; Hussain 等人 2017）。

在上述大部分被审查的研究中，测试函数被视为代理问题，帮助开发者评估优化算法的收敛效果、解的稳定性以及扩展性（Brockhoff 等人 2015；Musafer 等人 2022；Venter 2010）。除了性能指标，开发者还可以检查运行时间和内存消耗（Liang 等人 2005）。全面的基准测试过程还可以揭示算法特性，如早熟收敛或容易陷入局部最优（Andrei 2008）。通过系统地分析合成测试的性能，可以更好地预测算法在实际任务中的表现，并通过观察算法在不同函数上的行为来分离促进或阻碍收敛的因素（Dieterich 和 Hartke 2012）。

测试函数的作用还在于确立竞争方法之间的可比性（Olson 等人 2017；Chen 等人 2022）。当不同研究组在相同的基准套件上评估其算法时，同行比较成为可能（Hoffmann 等人 2019）。此外，测试函数通常具有教学目的，新接触优化的学生和从业者可以使用它们来掌握调优策略或局部搜索步骤。因此，测试函数有助于弥合理论与应用之间的差距，并逐步建立算法如何响应不同问题特征的直觉（Kazemzadeh Azad 和 Kazemzadeh Azad2023）。

我们从技术定义和详细内容的目录开始，这将作为本综述的入门指南。鼓励读者查阅其他资料以获得更全面的讨论（Cavazzuti 2013; Diwekar 2021; Pedregal 2004; Adby 2013）。

3.1 基准函数

基准函数是一个定义明确、构造良好的数学表达式，用于在受控条件下系统地评估优化算法的有效性。此类函数通常包含已知属性，如其全局极值的位置和值，并设计用来模拟实际优化任务中遇到的复杂性。这些复杂性可能包括陡峭的梯度、不连续性、欺骗性的局部最小值、平台和狭窄的吸引盆地。

3.2 连续性

一个函数 f: ℝn→ ℝ 被称为在点 x0∈ ℝn 处连续，如果对于每一个 ε > 0，都存在一个 δ > 0，使得 ‖x − x0‖ < δ 蕴含 |f(x__) − f(x0)| < ε。这个定义反映了能够无突然跳跃或间断地描绘 f 的图像的直观概念（Terazono 和 Matani__ 2015）。连续性保证了输入的微小扰动会产生相应微小的输出变化。这一特性在许多优化场景中至关重要，其中基于梯度的方法或局部搜索方法依赖于平滑过渡来定位极值。

3.3 收敛性

收敛性描述了迭代优化算法在向最优解进展时的行为（Bull 2011）。更精确地说，如果对于每个 ε > 0，都存在一个迭代次数 N，使得对于所有 n ≥ N，第 n 次解 xn 与 x* 之间的距离小于 ε，则称该算法收敛到点 x*。快速收敛速度通常是可取的，但过于激进的方法可能导致早熟收敛，即算法在没有充分采样搜索空间的情况下就停留在次优局部最小值。

3.4 可微性

一个定义在 nℝ上的实值函数 f 在点 x0 处可微，如果存在一个线性映射 A: ℝn→ ℝ，使得 lim(x → x0) (|f(x) − f(x0) − A(x − x0)|/‖x − x0‖) = 0。在一维情况下，这个条件简化为导数的存在，而在多维情况下，它涉及偏导数（Sachs 1978）。可微性对于使用基于梯度的方法至关重要，因为导数既指示了最陡峭上升（或下降）的斜率和方向，从而能够更直接地找到局部最小值或最大值。

3.5 维度

维度是指函数或问题空间中独立变量或自由度的数量。在一个 n 维问题中，每个可行解可以表示为 ℝn 中的一个点，其体积随 n 的增加呈指数增长。高维搜索空间通常难以导航，因为许多优化技术会遭受“维度灾难”的影响，即计算或采样工作随维度的增加而显著增加。

3.6 适应度景观

适应度景观是一种概念上的地形，其中搜索空间中的每个点都映射到一个表示其质量或适应度的实数值（Zou 等人 2022）。该空间中的峰、谷、脊和高原表示高绩效和低绩效区域，指导搜索启发式算法的轨迹。可视化或建模适应度景观通常能提供对底层函数复杂性和试图发现全局最优解的算法行为的洞察。

3.7 全局最优

搜索空间中的一个点 x* 如果对于所有 x（在最小化情况下）满足 f(x) ≤ f(x) 或（在最大化情况下）满足 f(x) ≥ f(x)，则该点为全局最优解。在许多现实世界和理论情境中，定位全局最优解具有挑战性，因为景观通常包含多个吸引域和欺骗性的局部最优解。寻求全局解的算法通常会包含机制（例如，随机采样或多起点技术），旨在避免陷入非全局解。

3.8 局部最优解

一个点 x0 被称为局部最优，如果在其周围存在某个邻域，使得 x0 产生最佳函数值。形式上，如果存在一个半径 δ > 0，使得对于所有满足 ‖x − x0‖ < δ 的 x，都有 f(x0) ≤ f(x)，则 x0 是局部最小值。与全局最优不同，局部最优仅优于其直接邻域内的点。在高维或多模态景观中存在多个最优值会为搜索算法引入额外的复杂性，这些算法可能需要专门的机制来逃离局部最优区域。

3.9 模态

模态描述了一个函数所具有的不同极值（例如，最大值或最小值）的数量。单峰函数只有一个主导的极值，而多峰函数表现出多个极值。更高的模态通常会增加优化任务的难度，因为算法必须正确地在多个峰或谷之间导航以找到全局解。因此，多峰函数在基准测试中非常有价值，因为它们严格测试了算法对早熟收敛的抵抗能力和探索搜索空间不同区域的能力。

3.10 噪声和随机性

噪声代表不可预测或随机的波动，可能破坏函数的真实评估值，通常源于测量误差、模型参数的不确定性或数值不精确（Nissen and Propach 1998）。对于函数 f(x)，观测值可能是 f(x) + ε，其中 ε 是一个随机变量。评估过程中的随机性可能误导确定性搜索步骤，因此需要鲁棒的优化策略来区分真实改进与随机扰动。在实践中，噪声抗性通常涉及统计采样、平滑技术或自适应参数设置，这些方法能够容忍波动的适应度分数。

3.11 优化算法

优化算法是一种为在指定域内找到给定目标函数的最佳解而设计的计算过程。此类算法可能利用确定性规则（例如，梯度下降、牛顿法和分支定界法）或随机启发式方法（例如，进化策略、模拟退火和群体智能）。核心组件通常包括初始化方案、迭代更新步骤和终止标准。其有效性通过算法定位全局最优解的优劣和速度来衡量，同时还要考虑在各种问题约束下保持稳定性和可扩展性。

3.12 可扩展性

可扩展性表示算法或函数在扩展到更高维度或更大问题实例时，如何保持其定义属性的一致性（Maltese 等人 2018）。对于基准函数，可扩展性是指在维度 n 增加时，保持其特征特性（如模态、可分离性或平滑性）。对于算法，它关注的是随着问题规模的增长，性能是逐渐下降还是急剧恶化。可扩展性测试问题可以揭示优化方法在高维度或参数扩展中的潜在局限性。

3.13 搜索空间

搜索空间包含优化算法操作的候选解集的完整集合。形式上，如果函数域是 D ⊆ ℝn，那么每个 x ∈ D 都是一个可能的解。这个空间的几何形状和边界，例如凸性或连通性，可以极大地影响算法性能。约束问题通过线性或非线性不等式限制 D，通过限制可行区域并可能将空间划分为多个不相交的子集来增加复杂性。

3.14 可分离性

一个函数 f(x1, x2, …, xn) 是可分离的，如果它可以表示为独立函数的和，每个函数只涉及一个变量：f(x1, x2, …, xn) = g1(x1) + g2(x2) + … + gn(xn) (Hochbaum and Shanthikumar 1990). 当这个性质成立时，原始的 n 维问题可以分解为 n 个独立的单维问题，极大地简化了分析和计算工作。许多优化技术利用可分离性来降低搜索复杂度，实现更快的收敛和更直接的梯度或直接搜索方案。

4.1.1 连续与离散

优化文献中的许多基准函数属于连续类别（Stein 等人 2004）。这类函数在实值变量上定义搜索空间。具有光滑表面的经典连续基准函数，突出了基于梯度或无梯度的优化器如何处理局部最小值或狭窄盆地等问题（Suwandi 2022）。连续函数还允许对维度和边界约束进行精细控制，这使得它们在测试梯度下降变体或进化策略时非常有价值。连续函数的例子包括 Rosenbrock 函数，它有一个狭窄的弯曲山谷，使得简单算法难以收敛。Rastrigin 函数包含多个规则分布的局部最小值，测试算法的全局探索能力。Schwefel 函数通过将其放置在定义域边界上，设计来诱使解偏离全局最优值，突出了在探索与利用之间取得平衡的难度。

相比之下，离散基准将域限制为组合配置（Tilahun 和 Ngnotchouye 2017）。例如包括旅行商问题（TSP）的变体或基于整数的背包公式（Pop 等 2024）。与连续曲面不同，离散景观由可行解之间的离散跳跃组成。这种结构差异通常使基于梯度的方法不适用，并强调了元启发式或专门整数规划技术的作用。因此，离散函数测试算法在有效探索和利用组合邻域方面的能力，这使得它们对于评估基于群体的算法或专门的分支定界策略（Kaji 2021）至关重要。例如，在 TSP 中，挑战在于通过一组城市找到最小的汉密尔顿回路，通常由成对距离定义（Wang 2014）。 这个问题表现出可能的排列组合呈组合式增长，这可以用来衡量算法在解的质量和计算工作量方面的性能。一个离散函数的例子是二次分配问题（QAP），它具有组合式复杂性（Loiola 等人 2007）。QAP 的核心是将一组设施分配到位置以最小化交互成本。QAP 的搜索空间随问题规模呈阶乘式增长，这意味着元启发式算法或专门的整数规划方法需要仔细平衡强化和多样化。这些任务不依赖于平滑的梯度信号；相反，它们推动基于群体或局部搜索的启发式算法去发现离散元素近乎最优的排列。

多模态测试函数用于检验优化算法中的多样化、探索和局部搜索改进等算法特性。例如，主要依赖局部梯度信息的算法可能会收敛到次优局部解，除非它们结合了重启、动量或自适应步长方法，从而能够摆脱局部陷阱。与此同时，遗传算法、粒子群优化等基于群体的算法，如果保持足够的种群多样性，可以更有效地扫描广阔的搜索空间，这有助于防止过早收敛到单一局部最优解。

4.1.4 可微与不可微

基准函数和测试函数也可以根据是否可微进行分类（Kiseleva 和 Stepanchuk 2003）。经典的可微函数，如 Sphere 函数、Rosenbrock 函数和 Ackley 函数，通常作为基于梯度的方法的测试平台，因为算法依赖于梯度信息来指导搜索（Daoud 等人 2023）。在这些情况下，可微性允许更精确的步骤：梯度提供了局部曲率信息，高阶方法可以利用 Hessian 近似来加速收敛。此外，可微函数支持多种高级理论分析，如局部收敛速度和临界点的稳定性。

不可微函数可能源于绝对值项、分段定义或离散转换（Zhou and Hu 2014）。一个经典例子是一维绝对值函数 f(x) = ∣x∣，它在 x = 0 处不可微。在更高维度中，包含 max 或 min 操作或模拟工程系统中切换行为的函数，在其景观中表现出角点或边缘，从而破坏了可微性。这些不连续性可能出现在实际情境中，例如某些材料设计任务中的摩擦模型。这些基准函数需要替代的优化技术，这些技术不依赖于平滑梯度，因此元启发式算法可能具有吸引力，因为它们通常对非平滑性具有鲁棒性，并且通常通过基于群体的采样（而非基于梯度的移动）来探索搜索空间（Cuevas et al. 2024）。 例如，像 Step 函数或 Schwefel 问题变体这样的函数可能包含平坦的平台和尖锐的不连续性，这突出了基于梯度的方法的局限性，同时也在测试元启发式的适应性。值得注意的是，在某些情况下，非可微性是通过约束有意引入的，这些约束会转化为惩罚函数。在这种情况下，如果惩罚函数被选作指标或基于步进的度量，它会在可行域和不可行域之间的边界处创建一个不可微区域（Shor 等人，2012）。

4.1.5 可扩展性与非可扩展性

基准函数中的可扩展性指的是在不改变问题定性特征的情况下调整函数大小或维度的能力（Durillo, Nebro, Coello, 等 2010）。可扩展函数是指允许随着问题维度的增加系统地探索算法性能的函数。例如，上述提到的 Sphere 函数是显然可扩展的，开发者可以在保持其凸性和二次轮廓的同时添加或删除维度。类似地，Ackley、Griewank 和 Rastrigin 这类函数具有众所周知的 d 维通用形式，即使维度从 2 增加到数百或数千个变量时，仍能保留多个局部最小值或高度崎岖等关键特征（Mahdavi 等人 2015）。

相比之下，不可扩展函数仅限于固定规模的问题，或在扩展到更高维度时失去其代表性（ECAI 2006）。某些经典的工程基准问题，如特定的桁架设计或齿轮传动优化问题，是用一套固定的参数和约束定义的，这些参数和约束是为特定应用量身定制的（Öztürk and Kahraman 2023; Ben Younes et al. 2022）。试图扩展这些基准问题可能会引入人工变量或约束，而这些变量或约束不再反映原始问题的物理/实际背景。类似地，离散问题可能无法以有意义的方式进行简单扩展。例如，虽然可以向旅行商问题（TSP）中添加更多城市，但所得实例在复杂性上可能会有显著差异，或无法保留原始配置的独特挑战。因此，平衡可扩展和不可扩展的基准问题可以促进全面的分析。 可扩展函数揭示了算法如何扩展到大规模或高维环境，而非可扩展基准测试则提供了对真实复杂性的保真度。

4.1.6 约束与无约束

基准函数可能包含对变量空间的显式约束，以反映现实世界问题中的物理、资源或监管限制（Venkata Rao 2016; Lagaros 等人 2023）。约束优化要求算法在满足约束条件的可行区域内进行搜索，通常通过惩罚函数、可行性修复启发式方法或多阶段搜索策略等方法实现。这些约束可以是线性的或非线性的，严格限制允许的搜索域或微妙地引导解沿着复杂的流形。无约束函数省略此类条件，默认情况下允许探索整个搜索空间（Nocedal 1992）。虽然无约束基准为测试算法的原始优化能力提供了基础环境，但它们并不能完全捕捉实际应用的复杂性（Cui 等人 2017）。 因此，将约束与其他挑战性属性（如多模态或高维性）结合，有助于更好地评估算法在可行区域变窄或碎片化时能否保持收敛质量。

除了显式约束外，实际问题的表述往往还包含隐式约束（即那些并非直接以方程式或不等式形式表达，而是源于实际考虑的限制，例如计算预算或实时响应能力）（Hellwig 和 Beyer2019）。值得注意的是，这类函数在某些进化计算竞赛（CEC）中可见（Molina 等人 2018）。寻求模拟这些场景的研究人员/开发者可能会添加资源约束、动态约束或时间敏感约束，这些约束迫使算法在严格的操作窗口内找到可行解（Gupta 等人 2021）。

4.1.7 凸与凹

另一个重要区别是凸基准与非凸基准之间的差异（Yang 2010）。像著名的 Sphere 函数或简单的二次碗这样的凸函数只有一个全局最小值，没有局部最优值。这类函数为基于梯度的方法提供了相对直接的寻优路径。相比之下，Ackley、Rastrigin 或 Griewank 等非凸函数具有多个局部最小值或复杂的拓扑结构，这对任何单模态搜索方法都构成挑战。这些非凸函数更好地反映了现实世界机器学习优化任务的复杂性，其中可能存在局部陷阱、平坦区域和高度弯曲的表面。

4.1.8 单目标与多目标及多目标

单目标函数，即优化单一性能指标，仍然是许多基础研究中的默认选择（Bonyadi 和 Michalewicz 2017）。专注于单一成本或适应度函数的简单性，便于清晰比较收敛速度、解的准确性和稳定性。因此，开发者可以追踪算法是否找到全局最优解、需要多少次函数评估，以及解对初始化或参数调整的敏感度。一些单目标函数的例子可以在连续函数列表中找到。

另一方面，多目标与许多目标基准测试呈现两个或多个必须同时优化的目标（Gunantara 2018）。这些目标可能存在冲突，因此需要算法在不同程度上权衡目标，这通常通过帕累托最优性的概念来体现。流行的多目标测试套件包含具有已知帕累托前沿的函数，使开发者能够评估算法发现多样化非支配解的能力。此类基准测试挑战元启发式的探索与开发机制，因为平衡多个目标需要将全局搜索与局部优化谨慎结合。它们还提供了评估超体积或分布等指标的机会，揭示算法如何有效逼近并沿真实帕累托前沿分布解。一些例子包括 ZDT、DTLZ 和 WFG 系列，或基于梯度的方法，这些方法利用基于帕累托的选择（Yalaoui 等人 2021；Deb 等人 2006）。 例如，ZDT1 提供了一个凸的帕累托前沿，相对容易捕捉，而 ZDT4 在决策空间中引入了多模态性。DTLZ 函数允许控制问题的维度、帕累托前沿的形状以及局部前沿的存在。这些测试集分别评估了多目标性能的不同方面，从收敛速度到解的多样性。

4.2 应用领域

虽然函数类型从高层视角展示了搜索空间的差异，但应用领域的分类反映了这些函数最相关的背景。以下将阐述其在机器学习、工程、经济和运筹学中的应用。类似地，这一讨论也可以扩展到其他领域。

4.2.1 机器学习

在机器学习（ML）领域，基准函数通常模拟模型损失景观或超参数调优目标（杨和沙米 2020）。这些函数从简化的典型误差表面表示（例如，二次碗）到更复杂的近似神经网络训练场景的景观（阿格拉瓦尔 2020）。机器学习中的合成基准可能引入非凸性、高维度或噪声梯度估计等特性，以模拟在真实数据上训练大型模型时遇到的实际困难。虽然精确建模此类表面在计算上是不可行的，但简化版本表现出如平台区域等特征，这些特征可能源于广泛的参数对称性或梯度信息较弱的鞍点。例如，某些专门设计的合成网络附带的目标函数形状类似于典型的前馈架构。人们可以使用类似的函数来评估优化器在避免鞍点平台和处理随机梯度噪声方面的性能。

4.2.2 工程、经济与运筹学
工程设计、资源分配或生产调度等领域需要包含可行性条件、多重性能指标或随时间变化的动态约束的基准函数。例如，在工程领域，基准测试可能模拟在材料属性、重量或成本约束下的结构组件应力（Gandomi 和 Yang 2011; Pereira 等人 2022）。例如，一个代表性函数可能将应力极限约束编码为高度非线性的边界，在搜索最小材料使用量的可行区域时必须遵守这些边界。另一个常见例子是设计桁架结构的横截面积，在应力约束下实现最小重量（Pereira 等人 2022）。
经济基准可以模拟供需均衡或平衡风险与回报的投资组合（Intriligator 2002）。运筹学通常涉及离散或混合整数基准，用于测试调度或路径决策（David 等人 2015）。例如，流水车间调度问题可以表述为一个优化函数，以最小化最大完工时间（一批工作的总完成时间）。约束条件是每个工作必须按指定顺序通过机器，导致问题规模增大时出现组合爆炸。一个著名的变体是排列流水车间调度问题，它通过要求所有工作遵循相同的生产行程来显著收紧约束（Liang 等人 2022）。

4.3.1 低维性

低维基准测试，通常在二维或三维空间中，作为视觉分析和教学理解的示例。开发者可以绘制等高线或三维曲面，以检查算法的轨迹，并直观地分析收敛现象。尽管这些函数很简单，但如果它们包含崎岖区域或陡峭梯度，仍然可能具有挑战性。然而，低维基准测试往往无法充分代表现代优化任务的规模。

4.3.2 高维性

随着维度的增加，搜索空间呈指数级扩展。基于群体的算法可能难以维持多样性，而基于梯度的方法可能面临噪声梯度或梯度消失的问题。在机器学习场景中，高维函数可以模拟深度神经网络训练相关的目标表面，这些表面通常具有大量局部最优值和尖锐的山脊。设计良好的高维基准不仅包含变量数量的增加，还包含诸如相关参数或部分可分性等现实特性。对于高维基准，将这些函数扩展到 n 维（例如 d 维罗森布洛克函数或 d 维拉斯特金函数）展示了局部最优值和狭窄山谷如何在搜索空间中迅速增多。与此同时，椭圆函数（二次碗的变体）由于主轴上非常不同的缩放因子，在高维度下会变得极其困难，因此需要自适应步长或高级协方差矩阵自适应。 某些竞赛（如 CEC 基准套件）中的混合组合（HC）函数将多个非凸函数组合成一个高维问题，通过混合不同的局部表面结构，更好地反映了复杂的现实世界条件（Mohamed 等人 2023）。

4.3.3 动态或时变

动态基准会随时间修改目标函数或约束条件的特征，迫使算法适应（Xiang 等人 2022）。这些变化可能是周期性的或不可预测的，以反映现实世界中数据流演变或环境条件变化的情况（Yazdani、Nouhi 等人 2023）。例如，一个动态函数可能会将其全局最优解移动到搜索空间的不同位置，改变约束边界，或引入新的局部最优解。例如，一个给定的函数可能沿着预定义路径平滑地移动其全局最优解，如果算法检测到这种变化模式，就能预测并跟踪移动的最优解。其他动态基准可能会突然重新定位最优解，迫使算法快速重新探索搜索空间。一个著名的动态基准是移动峰基准（MPB），它随着时间推移改变搜索空间中多个峰的位置和高度（Moser 和 Chiong 2013；Fox 等人 2022）。 适用于动态优化的算法必须平衡先前解的记忆与持续探索，以展示在跟踪移动目标时的敏捷性。

4.3.4 计算开销

在计算开销方面，某些基准测试包含用于函数评估的复杂模拟或数值积分（Valiante 等人 2021）。例如，气候建模中的单次函数评估可能需要运行一个偏微分方程求解器或详细的有限元分析，这可能会非常昂贵。在这样函数上测试的元启发式算法或机器学习算法可以强调计算效率、并行化或基于代理的建模。这个计算方面对于实际应用至关重要，因为一个具有强理论收敛性但评估需求高的算法可能在时间敏感的工程或机器学习任务中无法使用（Lan 等人 2022）。

7.挑战、局限性和未来方向

尽管在设计和构建基准测试方面持续努力，优化研究仍面临重大挑战（Alba 等人 2013；Floudas 等人 2004）。本节探讨了开发者在使用基准测试函数时遇到的常见陷阱（挑战），并阐明了未来的研究方向。

首先，选择性报告可能导致性能声明无法推广到所选测试函数之外（Sarhani 等人，2023）。因此，过拟合成为一个实际风险，因为它无法应对多样化的障碍。另一个挑战是缺乏标准化（Velasco 等人，2024）。参数设置差异即使在使用相同的测试函数时也可能产生矛盾的结果（Shadkam，2022）。这种不一致性使得直接比较变得复杂，并破坏了可重复性。此外，缺乏关于运行次数、初始条件、函数评估预算或统计测试的标准化规则，加剧了这一挑战。除了可重复性之外，过度强调经典函数可能导致技术改进停滞，因为这些函数通常依赖于特定的景观（并且可能无法捕捉离散性质）（Razumikhin，1987）。

一个显著的不足之处是复杂性的代表性不足（Bubeck 2015）。尽管一些专门的测试套件包含了动态元素/约束/噪声评估，但这些特征通常仍然相对简化，并不能复制时间序列的变化或异常。类似地，动态基准测试经常依赖于平滑演变的地形，即使现实场景可能经历急剧转变（Yazdani 等人 2022）。虽然存在一些成熟的动态套件，但未来的工作可能会增加它们的复杂性，以阐明算法如何应对重大干扰（Yazdani 等人 2021a, 2021b）。

大规模和高维基准测试仍然处理不均等（Esmin 等人，2015 年）。尽管研究人员可以将现有函数扩展到更高维度，但这种方法并不总是反映现实的参数相关性或数据依赖性。将基准测试套件扩展到包含真正高维域、具有相关变量、多尺度或区域特定约束的领域提出了另一个独特的挑战（Yang 等人，2018 年）。与此问题相关的另一个挑战是改进在高维度和不同尺度下如何测量性能

传统的指标，如最终解的精度或收敛速度，可能会掩盖探索中的潜在挑战。开发人员可以将特定问题的指标集成，以评估搜索过程如何识别关键变量周围的相关子空间（Wang 等人，2019 年）。

尽管目前的研究重点仍在于完善经典测试套件以及弥合合成任务与现实任务之间的差距，未来的研究方向应致力于扩展基准驱动研究的广度和深度。例如，量子计算已经开始影响优化领域，主要通过借鉴量子力学原理的量子启发式方法，而无需完全依赖量子硬件（Li 等人，2020）。这类算法有可能以更高的效率解决特定问题（Moll 等人，2018）。例如，将进化算子与量子原理相结合可以产生新的跳出局部最优的策略。在测试函数中捕捉这些协同效应需要独特的设计选择（Cano 等人，2017）。

目前，测试函数常被视为黑箱，对阐明哪些因素最影响解空间关注有限（Audet 2014；Alarie 等人 2021）。随着可解释性的兴起，未来的基准测试可包含可解释结构，使研究人员能够识别哪些变量或约束驱动了适应度景观的形状（Naser 2021）。这些函数可通过嵌入元信息或层级关系来帮助解释算法决策的依据。这种额外的透明度有助于更好地理解为何某些方法表现优异（Zhou 等人 2024）。例如，可解释函数可能会标记关键特征或记录特定参数在特定域区域如何变化影响全局最优（Fyvie 等人 2025）。这种补充性背景信息有助于诊断算法失效并提升泛化能力。

缓解上述许多挑战和局限性的策略主要围绕社区倡议，鼓励公开分享新的测试函数、数据集和代码库（Nejati 等，2023；Khari 等，2020）。这种方法将促进基准套件代表性方面的渐进式改进。因此，结合离散子问题和连续子问题的基准套件的共同进化可能突出混合优化算法的适应性（Kumar 等，2021；Van Thieu 和 Mirjalili，2023）。此外，容器化技术的兴起（例如，Docker（Combe 等，2016））可以帮助简化再现实验环境的过程（Casalicchio 和 Iannucci，2020）。此类社区驱动的努力还可能涉及对潜在新基准或现有基准修改的众包反馈。

在现有文献中，元启发式算法与数值测试问题或测试函数之间存在密切关系。通常，任何新元启发式算法的引入都会伴随着一套基准测试函数，这些函数用于证明算法的效率[1]。事实上，并不存在标准的测试函数列表。不同的研究人员选择不同的基准测试函数集合，而这一选择并没有任何合理的理由。此外，测试平台的函数数量也差异很大，从两个函数到二十多个函数不等（数量不等）。测试函数的范围包括不同的特征；例如，单峰和多峰、规则和不规则、可分离和不可分离等（特征类型不等）。这些函数也经常被旋转或平移，并且使用更广泛的维度（从 10D 到 1000D），以增加优化景观的复杂性。这给想要在基准测试函数上验证现有元启发式算法的任何修改版本或全新算法的新研究人员造成了显著差距。 通常，研究人员在景观结构、困难性、模式、可分离性、非线性以及其他优化问题的各个方面缺乏足够的背景知识。

  1. 基准函数与特性

基准测试函数本质上是以数学数值函数形式呈现的优化问题。这些函数通过一组最适宜的参数值进行优化，以帮助实现最佳解决方案，其中 D 是问题的维度。最佳解决方案隐藏在遍布问题景观的大量次优解中，这些景观具有不同数量和类型的山丘与山谷。任何优化算法，包括元启发式算法，都倾向于尽快找到最佳解决方案（但不保证一定能获得）。元启发式算法的效率通过其全局搜索能力和局部收敛能力来衡量。具有更好全局搜索能力的算法难以被困在次优（局部最小值或最大值）位置。同时，具有高效收敛能力的元启发式算法难以错过邻域内的任何最佳解决方案。

可以找到许多在线资源提供更详细的信息以及 MATLAB 和 R 语言函数代码，例如[3]、[4]和进化计算大会（CEC）[5]等。根据问题景观的特征（模态、盆地、山谷和可分离性、维度）[2]，测试函数可以分为以下类别：

a. 模态

模态定义了问题景观中的峰数。这些峰形成了局部最小值和全局最小值的位置。

单峰函数：这些函数有一个山谷和一个全局最小值位置，最佳解位于此处。这类函数被认为容易求解，但移动和旋转这些函数会增加难度。元启发式算法可以用于这些函数来评估局部搜索能力。

多模态函数：这类函数维持多个解，但真正的全局最优解只有一个。此类函数具有许多局部最小值位置，但只有一个真正的全局最小值。因此，任何元启发式算法都需要遍历整个地形才能找到真正的全局最优解。这些函数难以求解，适合测试任何算法的全局搜索能力。

b. 盆地

盆地是一个被高山环绕的陡峭下降区域。多模态函数包含的局部最小值池的数量多于全局最小值池。性能差的算法往往被困在局部最小值池中，无法找到全局最小值池。

c. 山谷

山谷是被山峰环绕的区域，其宽度较窄而长度较长。从根本上说，它是一个变化范围较小的区域。元启发式算法在这些区域中运行时间较长，因为局部搜索倾向于对这些位置进行详细搜索。单峰函数和多峰函数以不同的方式塑造这些山谷，并且频率也不同。

d. 可分离性

可分离性定义了优化函数变量的方法。单峰函数和多峰函数都可以是可分离的，也可以是不可分离的。

可分离函数：如果函数是可分离的，每个变量 xi 都可以独立优化。这类函数易于求解。

不可分离函数：如果函数是不可分离的，那么它的所有变量之间都有很强的相互关系，不能独立优化。这种函数相对较难解决。

e. 维度

该属性定义了搜索空间的大小。维度越大，搜索空间越大，这意味着次优位置的数量越多。维度较小的函数容易解决，并且大多数元启发式算法在这些函数上表现最佳。然而，对于真正的性能评估，函数需要具有很高的维度。

全局优化在现实世界中有着广泛的应用，包括离散和非离散两种性质。其中涉及化学应用，如分子和团簇的结构优化，工程问题，如部件设计，物流问题，如调度和路径规划，以及其他许多领域。尽管通常的实践发现，通用的全局优化算法通常比针对具体问题进行调优的特定版本效率低得多，但使用基准问题来评估全局优化方案的基本性能仍然是有意义的。即使在最近的一些此类测试示例[1–6]（从最近的文献中随机选取），通常也会采用某些标准基准函数，其隐含的（但未经测试的）假设是这些基准函数的难度大致与实际应用的难度相匹配。有些基准函数甚至被宣传为特别具有挑战性。我们在原子和分子簇结构优化的挑战性、现实领域开发了基于进化算法（EA）的全局优化策略[7–12]。然而，当我们把我们的算法应用于那些传统的、抽象的基准函数时，这两种说法（挑战性，以及与现实应用相似性）均站不住脚。事实上，类似的怀疑早已被提出。例如，早在 1996 年 Whitley 等人[13]就认为，由于对称性和可分离性等固有特性，许多标准基准函数应该相对容易求解。有些函数甚至随着函数维度的增加而变得更容易。尽管如此，正如上一段引用所示，同一组传统的基准函数至今仍被不同领域的大多数研究人员不加区分地使用。 因此，通过本文，我们重新强调从实际角度出发那些早期的研究成果，增加其他测试函数，并将测试扩展到更高维度。此外，我们强调传统基准问题似乎过于简单，无法进行算法或实现细节之间的有效比较，并且它们不能得出关于全局优化算法在实际生活中的性能的结论。我们表明，使用不同种类的基准函数时，后者能够更好地实现。

在这些背景下，理论考虑通常集中于将问题分类为 N 类或 NP 类[14,15]，或评估边缘不同的参数表示[13,16]，或已知测试函数的混合组合[13]。此外，通常会在常规基准函数中添加转换、旋转以及随机噪声[6]。尽管这是一种可能使基准复杂化的可行方法，但通过引入附加参数和由此产生的兼容性问题，它限制了不同基准结果的可比性。鉴于在讨论独立开发的新算法时，这一点至关重要，因此本次讨论不包含以这种方式修改的基准函数。然而，无论涉及此类问题分类和算法特性，在大多数实际应用中，随问题维度（即要优化的参数数量）的扩展起着关键作用。化学结构优化集群就是一个明显的例子：实际中至关重要的现象包括集群聚集和碎片化，或性质对团簇尺寸的依赖，而隔离单一团簇尺寸是一项艰巨的实验挑战。因此，人们不是研究单一团簇尺寸，而是试图系统地研究一系列团簇[8, 9, 17–19]，仅受限于所拥有的最大计算能力。显然，最小规模的减少（例如从 O(N)到 O(N)）可能允许研究显著更大的团簇尺寸。

问题维度的规模与所需的全局优化步骤数量当然与全局优化算法的特性相关。对于进化算法，这包括交叉和变异算子、可能的局部优化步骤以及针对特定问题调整的附加算子。我们希望在此介绍我们最近开发的用于化学问题进化全局优化的框架 ogolem [11]中一些标准测试函数的最新结果。通过筛选解决高达 500（在一种情况下为 10000）维测试函数所需的全球最小化步骤数量，我们获得了不同交叉算子与这些函数维度规模的关系。此外，在某些情况下，我们比较了包含和不包含局部优化步骤的运行结果，以研究基于梯度的最小化对规模的影响。最后但同样重要的是，我们将这些标准测试函数上的性能与不同类型的测试函数上的性能进行比较，后者显然更具挑战性，在某些方面更接近实际问题。

当前工作有助于定义全球优化算法基准测试的新基线和标准。通过展示现代遗传算法实现解决所审查的基准函数时的扩展性，我们希望鼓励其他全球优化技术开发者不仅报告特定维度下定义的基准函数的结果，更要关注扩展行为，并将他们的结果与我们的经验基线进行比较。

  1. 结论

针对四种不同的标准基准函数进行了规模研究，并补充了对第五个函数的性能测试。使用没有特定问题成分的简单遗传算法技术，我们在所有这些情况中的观察行为与将相同技术应用于现实问题时（通常包括特定系统添加）的观察结果明显不同：所有基准测试都可以用次二次规模解决，而在实际应用中我们最多只能达到三次规模，并且通常不得不接受更差的结果。此外，基准测试通常不允许对不同的交叉算子的性能做出统计上显著的结论，也无法决定是否包含局部优化。因此，总体而言，这类基准测试似乎未能满足作为与实际应用相关的全局集群结构优化或类似领域的测试平台的目的。

我们将这些传统基准函数的行为与两种不同类型的函数进行了对比。其中一种类型是“景观生成器”类，这里展示了一个特别简单的实现，即由随机分布的高斯函数生成的搜索景观。通过改变高斯特性（深度、宽度、密度、维度），搜索空间特征可以任意调整，范围从“山脉”到“高尔夫球场”。此外，由于构成的高斯函数完全不相关，这类问题的难度本质上比传统基准函数更大，传统基准函数的最小值特性是按照简单规则构建的。这一点在我们的测试结果中得到了鲜明体现。

作为另一类基准函数，我们展示了欺骗型函数，其设计目的是误导全局优化。通过使全局搜索更加复杂，例如在基于群体的搜索中确保群体具有足够的多样性，可以显著降低这类函数的难度。在足够灵活的设置下，这类基准函数仅仅是景观生成器的子类。

未来的工作将需要验证我们的猜想：实际世界中的问题往往介于传统的、相对简单的基准函数和不太相关、更具欺骗性的基准函数之间，无论是在搜索空间特性上，还是在它们对全局优化算法提出的难度上。无论如何，我们希望已经有力地证明了，由于它们的简单性，传统的基准函数不应单独使用，既不能用于辅助全局优化算法开发，也不能用于评估性能。

广义而言，优化可以分为确定性优化算法和随机性优化算法[2]。确定性优化涉及一组预定的程序和步骤，旨在根据问题的内在性质和结构找到最佳解决方案。确定性优化方法的例子包括牛顿法、共轭梯度法和基于梯度的算法，如梯度下降算法。

另一方面，随机方法涉及利用随机性或概率技术来探索问题空间中的解决方案，具有灵活性和适应性。随机优化的例子包括进化算法、群智能和元启发式算法。这些方法在其搜索过程中引入随机性，允许探索更广泛的解决方案空间，并可能找到确定性方法可能忽略的新颖解决方案。

引言：

优化是一种强大的技术，用于为各种工程和规划问题获得合适的解决方案。在过去三十年中，众多研究人员开发了一系列元启发式算法，以解决来自工程、商业等领域的复杂优化问题。事实上，虽然基于梯度的经典优化算法通常无法处理非线性、非凸以及多模态问题，但元启发式算法能更有效地处理这些问题。当寻找最优解的方法因计算成本过高而无法应用时，它们也很有用（Bennis and Bhattacharjya, 2020）。这些元启发式算法通常是迭代优化算法，它们最常共享一个算法步骤，即解的初始化。研究界普遍认为这一步在优化过程中起着重要作用，因为之后生成的所有解在一定程度上都依赖于它们的前置解，并最终依赖于初始解或初始解集。然而，测量依赖程度并不简单。此外，在处理这个问题时，必须阐明方法的目的，以及它是否旨在提高质量与/或多样性。多样性是指解有效探索整个搜索空间的程度，而质量通常指生成的解与全局最优的接近程度。为简化起见，我们采用此定义来描述质量，并将在后面讨论。在搜索空间的一定限制内寻找提高质量的过程通常被称为强化（也称为开发），而增强多样性的过程通常被标记为多样化（或探索），其中搜索尝试探索解决方案空间中尚未访问的部分。在设计和改进元启发式算法时，平衡这两个目标是最具挑战性的问题。 然而，由于在理论上和定制化方面进行这方面工作存在困难（例如，Chen 等人，2009 年），它们之间的平衡尚未得到充分探索。本文的目标是弥合元启发式初始化的进展与上述权衡之间的差距。

通过分析元启发式初始化的文献，我们注意到很少有人以全面和系统的方式对初始化方法进行综合和分析。据我们所知，最近一篇关于元启发式初始化的调查是由 Kazimipour 等人于 2014 年提出的，该调查主要关注基于群体的元启发式方法。该论文的主要贡献是根据三个不同的维度对它们进行分类，这些维度是随机性、组合性和普遍性。与 Kazimipour 等人（2014）相比，我们的贡献在于对所提出的方法进行批判性分析，并强调处理该问题至关重要的主要相关主题，同时提供对其的见解。此外，我们不局限于基于群体的方法。事实上，为了妥善解决这些问题，有必要考虑主要的元启发式挑战，尤其是提到的困境。尽管如此，我们注意到 Kazimipour 等人（2014）引用的许多论文并没有提供真实证据来证明它们的初始化方法为何有效以及在何种条件下有效。

一、前期知识

什么是Hexo

  • Hexo 是一个用 JavaScript 编写、运行在 Node.js 环境中的静态博客框架

为什么用Hexo必须要用到Node.js

  • 因为 Hexo 不是一个独立的 exe 软件，也不是一个编译好的 Windows 桌面程序。它本质上是一套用 JavaScript 编写的程序。既然是 JavaScript 写的，而且不是浏览器里运行的前端页面代码，那它就需要一个”浏览器之外的 JavaScript 运行环境”来执行。这个运行环境就是 Node.js。Hexo 这个具体工具本身，就是用 Node.js 技术栈写出来并发布在 npm 生态里的。
  • Hexo 在工作时并不只是简单显示几篇文章，它会做一系列处理，这些处理都依赖 Node.js 环境来完成。比如：

你写的 Markdown 文章要被读取。
主题模板要被解析。
配置文件要被加载。
文章内容要被渲染成 HTML。
静态资源要被整理复制。
本地预览服务器要被启动。
插件系统要被加载。

这些动作本质上都是本地程序行为，不是浏览器自动帮你做的。Hexo 本身就是负责做这些事情的程序，而 Node.js 就是让这个程序能跑起来的底层运行环境。

为什么要部署到GitHub

什么是JavaScript

  • JavaScript 是一种编程语言。最直接地说，它是让计算机按照你写的规则去处理数据、执行逻辑、响应用户操作的一套语言体系。
  • HTML 负责页面有什么内容，CSS 负责页面长什么样，JavaScript 负责页面动起来、能交互、能处理逻辑。
  • 现在的 JavaScript 早就不只是在网页里用了。它还能写服务器程序、命令行工具、桌面应用、移动端应用，甚至能做一些自动化脚本和工程化工具。
  • 程序语言不是写出来就自动能跑，必须有一个执行环境。JavaScript 最经典的执行环境有两种。
    • 第一种是浏览器
      • 比如 Chrome、Edge、Firefox。这些浏览器内部都带有 JavaScript 引擎。你打开网页时，浏览器会读取网页里的 JavaScript 代码并执行它。所以网页中的点击事件、动画、表单校验之类的功能，都是浏览器在执行 JavaScript。
    • 第二种是 Node.js
      • Node.js 可以理解成一个让 JavaScript 脱离浏览器也能运行的环境。这样 JavaScript 就不再只能控制网页，还能在本地电脑或服务器上读写文件、启动服务、处理命令行参数、搭建后端程序。前面说 Hexo 依赖 Node.js，本质上就是因为 Hexo 是用 JavaScript 写的，但它不是在浏览器里运行，而是在本地命令行环境里运行，所以需要 Node.js 来执行。

什么是Node.js

  • Node.js 让 JavaScript 不只在浏览器里运行，也能在本地和服务器运行。

什么是Git

  • Git是与GitHub仓库进行交互的终端
